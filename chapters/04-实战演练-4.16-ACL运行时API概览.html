4.12 ACL 运行时 API 概览

ACL（Ascend Computing Language）是昇腾 NPU 的运行时编程接口，类似于 CUDA 的 Runtime API。算子的 Host 侧代码通过 ACL 来管理设备、分配内存、控制执行流。本节介绍 ACL API 的整体分类和使用场景。

ACL 在算子开发中的角色

在 Ascend C 算子开发中，Kernel 侧代码运行在 NPU 上，负责实际计算；Host 侧代码运行在 CPU 上，负责准备数据、启动 Kernel、获取结果。ACL 就是 Host 侧与 NPU 交互的桥梁。

ACL API 分类

ACL API 按功能分为以下六大模块：

模块核心 API作用
初始化aclInit, aclFinalizeACL 运行时的初始化和销毁
设备管理aclrtSetDevice, aclrtResetDevice选择和管理 NPU 设备
内存管理aclrtMalloc, aclrtMemcpy, aclrtFreeDevice 内存的分配、拷贝和释放
流管理aclrtCreateStream, aclrtSynchronizeStream异步执行流的管理
事件管理aclrtCreateEvent, aclrtRecordEvent流间同步和事件记录
错误处理aclGetRecentErrMsg获取最近的错误信息

基础 API vs 算子调用 API

ACL 提供的 API 可以分为两类：

基础运行时 API

这些 API 管理 NPU 的运行环境，与具体算子无关：

  • 初始化/销毁：aclInit / aclFinalize — 程序生命周期管理
  • 设备管理：aclrtSetDevice / aclrtResetDevice — 选择哪张 NPU 卡
  • 内存管理：aclrtMalloc / aclrtMemcpy / aclrtFree — 在 Device HBM 上分配和搬运数据
  • 流与事件：aclrtCreateStream / aclrtCreateEvent — 控制异步执行和同步

算子调用 API

这些 API 直接用于启动算子 Kernel：

  • 直调方式：my_kernel<<<coreNum, nullptr, stream>>>(args...) — 通过 <<<>>> 语法直接调用 Kernel 函数
  • 框架调用：通过 aclopCompileAndExecute 等 API 调用框架注册的算子

在 Ascend C 直调开发中，我们主要使用基础运行时 API + 直调语法的组合。

常用数据结构和枚举

在使用 ACL API 之前，了解以下基础类型定义有助于理解后续章节：

aclError（返回值类型）

typedef int32_t aclError;
#define ACL_SUCCESS 0

所有 ACL API 都返回 aclError，ACL_SUCCESS（0）表示成功，其他值表示各种错误。

aclrtMemcpyKind（拷贝方向）

typedef enum {
    ACL_MEMCPY_HOST_TO_HOST     = 0,
    ACL_MEMCPY_HOST_TO_DEVICE   = 1,
    ACL_MEMCPY_DEVICE_TO_DEVICE = 2,
    ACL_MEMCPY_DEVICE_TO_HOST   = 3
} aclrtMemcpyKind;

aclrtMemMallocPolicy（内存分配策略）

typedef enum {
    ACL_MEM_MALLOC_HUGE_FIRST  = 0,  // 优先 Huge Page
    ACL_MEM_MALLOC_HUGE_ONLY   = 1,  // 仅 Huge Page
    ACL_MEM_MALLOC_NORMAL_ONLY = 2   // 仅普通页
} aclrtMemMallocPolicy;

aclrtStream / aclrtEvent（不透明句柄）

typedef void* aclrtStream;
typedef void* aclrtEvent;

Stream 和 Event 都是不透明指针（opaque handle），你不需要知道它们的内部结构，只需要通过 ACL API 来创建、使用和销毁。

Q&A

Q：ACL 和 CUDA Runtime API 有什么区别？

A：两者在概念上非常相似——都是 Host 侧管理 Device 的编程接口。主要区别在于：ACL 针对昇腾 NPU 的硬件特性设计（如 AI Core、HBM、多核并行），而 CUDA 针对 NVIDIA GPU。API 命名和参数有差异，但"初始化→选设备→分配内存→拷贝数据→启动 Kernel→同步→取回结果"的整体流程是一致的。

Q：直调开发和框架开发在 ACL 使用上有什么不同？

A：基础运行时 API（初始化、设备管理、内存管理、流管理）在两种方式中完全相同。区别在于算子调用方式：直调使用 <<<>>> 语法直接调用 Kernel 函数，框架开发则通过 aclopCompileAndExecute 等接口间接调用。本系列教程聚焦直调方式。
