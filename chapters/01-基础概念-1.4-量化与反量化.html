1.4 量化与反量化

1.4.1 量化（Quantization）

量化是将高精度数据（如 FP32）映射到低精度数据（如 INT8）的技术。目的是减小模型体积、加速推理。

类比一下：一幅 2400 万像素的照片（FP32）占 70MB，压缩成 JPEG（INT8）只占 3MB，虽然损失了一些细节，但人眼看起来差别不大。

1.4.1.1 量化公式

q = round(x / scale) + zero_point

其中 scale（缩放因子）的计算方式为：

scale = (max_val - min_val) / (qmax - qmin)

max_val 和 min_val 是原始数据的范围，qmax 和 qmin 是目标量化类型的范围（如 INT8 是 -128 到 127）。

计算示例：假设一组 FP32 权重值的范围是 [-1.0, 1.0]，要量化为 INT8（范围 -128 到 127）：

scale = (1.0 - (-1.0)) / (127 - (-128)) = 2.0 / 255 ≈ 0.00784
若 x = 0.5：q = round(0.5 / 0.00784) + 0 = round(63.78) = 64
若 x = -0.3：q = round(-0.3 / 0.00784) + 0 = round(-38.27) = -38

1.4.1.2 对称 vs 非对称

类型zero_point量化范围适用场景
对称量化0[-max, max]权重（通常以 0 为中心对称分布）
非对称量化≠0[min, max]激活值（ReLU 输出全为非负，不对称）

对称量化更简单高效（不需要处理 zero_point），但要求数据分布以 0 为中心。非对称量化更灵活，但计算时需要额外处理 zero_point 的偏移。

1.4.1.3 按粒度分类

量化的"粒度"指的是多少元素共享一组 scale 和 zero_point：

  • Per-tensor：整个张量共享一组 scale/zero_point。最粗糙，精度损失最大，但实现最简单。
  • Per-channel：每个输出通道一组 scale/zero_point。精度更好，是权重量化的常用方案。
  • Per-group：每 G 个元素一组 scale/zero_point（如 G=128）。精度最好，是大模型量化的主流方案（如 GPTQ、AWQ 都用分组量化）。

Q：Per-group 量化的 G 值怎么选？越小精度越好，为什么不直接用 G=1？

A：G=1 意味着每个元素都有自己的 scale 和 zero_point，那就等于没有量化——每个数还是要额外存一个 FP32 的 scale，体积反而更大了。实际工程中 G 的选择是精度和效率的平衡：G=128 是大模型量化的主流选择（GPTQ、AWQ），它在精度损失很小的同时，128 个元素共享一个 scale 的开销可以忽略不计。G=32 精度更好但开销翻倍，G=64 是两者之间的折中。

1.4.2 反量化（Dequantization）

反量化是量化的逆过程——把低精度数据还原回高精度：

x_approx = (q - zero_point) × scale

注意这里写的是 x_approx 而不是 x，因为反量化是有损的——量化时 round() 操作引入了舍入误差，反量化无法恢复原始值。精度损失的大小取决于量化方案：粒度越细（Per-group < Per-channel < Per-tensor），损失越小。

计算示例：接上面的例子，已知 INT8 量化值 q = 64，zero_point = 0，scale = 0.00784：

x_approx = (64 - 0) × 0.00784 = 0.50176

原始值 x = 0.5，反量化后得到 0.50176，误差约 0.002——这就是量化引入的舍入误差。

在实际推理流程中，量化和反量化经常成对出现：输入用 INT8 计算（快），输出反量化回 FP16/FP32（准）。

1.4.3 伪量化（Fake Quantization）

伪量化是一种训练时模拟量化误差的技术，也叫"量化感知训练（Quantization-Aware Training, QAT）"。

核心思想：在训练阶段，前向传播时做一次"量化→反量化"（模拟推理时的精度损失），让模型在训练过程中就学会适应这种噪声；反向传播时，梯度直接穿过量化操作（因为 round() 函数不可导，用 Straight-Through Estimator 近似为恒等映射）。

这样训练出来的模型，在真正部署为 INT8 推理时，精度损失会比"训练后直接量化（PTQ）"小很多。

Q：伪量化（QAT）和训练后量化（PTQ）有什么区别？什么时候该用哪个？

A：PTQ（Post-Training Quantization）是模型训练完成后，直接拿 FP32 权重做量化，不需要重新训练。优点是简单快速（几分钟搞定），缺点是精度损失可能较大。QAT（Quantization-Aware Training）是在训练过程中就模拟量化误差，让模型学会适应，精度损失更小，但需要重新训练（耗时较长）。实际工程中通常先试 PTQ，如果精度不达标再上 QAT。

Q：反向传播时 round() 函数不可导，STE 是怎么处理的？

A：STE（Straight-Through Estimator）的做法很简单粗暴——反向传播时假装 round() 不存在，把梯度直接传过去（相当于把 round() 当作恒等函数 y=x 来求导，dy/dx = 1）。这在数学上并不严格，但实践中效果很好——模型能"感知"到量化噪声的存在并学会适应，同时梯度更新不会因为 round() 的不可导而中断。
