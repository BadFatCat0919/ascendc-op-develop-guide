说明：

  • 指数位 + 尾数位 + 1（符号位）= 总位宽。例如 FP32：1 + 8 + 23 = 32 位 = 4 字节。
  • MXFP 类型（MXFP8、MXFP4）采用分组共享 Scale 机制：每 32 个数据元素共享 1 个 Scale 因子，因此实际存储开销 = 数据位宽 + Scale 位宽 / 32。
  • HiF8 是华为自定义的 8-bit 浮点格式，指数位和尾数位可配置，适配不同的精度-范围需求。
  • A5 新增类型（FP8、MXFP8、MXFP4、HiF8）主要面向大模型推理场景的量化需求，A5 的 Vector 和 Cube 单元原生支持这些类型的计算指令。

对齐粒度速算

DataCopy 每次搬运的数据长度必须是 32 字节的整数倍。各数据类型的最少搬运元素数：

数据类型字节数对齐粒度（最少元素数）计算方式

FP16 / BF16 / INT1621632 / 2 = 16
FP8 / INT8 / UINT813232 / 1 = 32
