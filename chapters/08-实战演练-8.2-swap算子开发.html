8.2 swap 算子开发（直调方式）

本章以 BLAS Level 1 的 sswap 算子为例，从零开始讲解一个完整算子的开发过程。swap 使用直调（direct-call）方式，即 Host 侧直接通过 <<<>>> 语法启动 Kernel，无需经过 aclnn 框架注册。这是 ops-blas 仓中 Level 1 算子的标准做法。

8.2.1 功能说明

BLAS sswap 的功能是交换两个单精度浮点向量的对应元素：

输入:  x = [x0, x1, x2, ..., x_{n-1}]
       y = [y0, y1, y2, ..., y_{n-1}]
输出:  x' = [y0, y1, y2, ..., y_{n-1}]
       y' = [x0, x1, x2, ..., x_{n-1}]

BLAS 接口签名为：

void sswap(int n, float* x, int incx, float* y, int incy);

其中 incx 和 incy 是步长（stride）。本实现限定 incx = incy = 1（连续存储），这是最常见的场景。

8.2.2 文件结构

在 ops-blas 仓中，sswap 的 arch35 实现由 3 个文件组成：

blas/swap/sswap/arch35/
├── sswap_tiling_data.h   # TilingData 结构体（Host/Kernel 共用）
├── sswap_kernel.cpp       # Kernel 侧：AscendC 类 + 入口函数
└── sswap_host.cpp         # Host 侧：参数校验 + Tiling + Kernel 启动
文件职责运行位置
sswap_tiling_data.h定义 Host 和 Kernel 共享的 Tiling 参数结构体编译期
sswap_kernel.cppAscendC Kernel 类，实现数据搬运和交换逻辑AI Core（Device）
sswap_host.cpp参数校验、Tiling 计算、内存管理、Kernel 启动CPU（Host）

8.2.3 TilingData 设计

TilingData 是 Host 和 Kernel 之间的通信契约。Host 侧计算好切分参数，通过 GM 传递给 Kernel。

#ifndef SSWAP_TILING_DATA_H
#define SSWAP_TILING_DATA_H

#include <cstdint>

struct SswapTilingData {
    uint32_t totalN;
    uint32_t perCoreN;
    uint32_t remainder;
    uint32_t tileSize;
};

constexpr uint32_t ELEMENTS_PER_BLOCK = 32 / sizeof(float);

#endif

各字段含义：

字段含义示例（n=10000, 4核）
totalN总元素个数10000
perCoreN每个核分配的元素数（向下对齐到 ELEMENTS_PER_BLOCK）2496
remainder尾部剩余元素，由最后一个核处理16
tileSizeUB 中单次迭代能处理的元素数（对齐到 ELEMENTS_PER_BLOCK）31744

ELEMENTS_PER_BLOCK = 32 / sizeof(float) = 8，因为 Ascend C 的 DataCopy 要求地址和长度都按 32 字节对齐。

8.2.4 Kernel 实现

Kernel 侧的代码运行在 AI Core 上。核心思路是：每个核从 GM 读入自己负责的那段 x 和 y 到 UB，然后交叉写回——x 的数据写到 y 的位置，y 的数据写到 x 的位置。

8.2.4.1 类定义

class SswapAIV {
public:
    __aicore__ inline SswapAIV() {}
    __aicore__ inline void Init(GM_ADDR x, GM_ADDR y, GM_ADDR tilingGm, TPipe* pipe);
    __aicore__ inline void Process();

private:
    __aicore__ inline void ParseTilingData(GM_ADDR tilingGm);
    __aicore__ inline void SingleIteration(uint32_t curOffset, uint32_t dataCount);

    TPipe* pipe_;
    GlobalTensor<float> xGM_;
    GlobalTensor<float> yGM_;
    TQueBind<QuePosition::VECIN, QuePosition::VECOUT, 1> inQueueX_;
    TQueBind<QuePosition::VECIN, QuePosition::VECOUT, 1> inQueueY_;
    SswapTilingData tiling_;
    uint32_t blockIdx_;
    uint32_t myOffset_;
    uint32_t myCount_;
};

关键成员说明：

  • TPipe* — 流水线管理器，负责 Buffer 分配（参见 4.3 内存管理）
  • GlobalTensor<float> — GM 上的全局张量，提供 operator[] 偏移访问
  • TQueBind<VECIN, VECOUT, 1> — 绑定输入和输出位置的队列，提供隐式同步（参见 4.6 同步与流水线）

8.2.4.2 Init：初始化

__aicore__ inline void SswapAIV::Init(GM_ADDR x, GM_ADDR y, GM_ADDR tilingGm, TPipe* pipe)
{
    pipe_ = pipe;
    blockIdx_ = GetBlockIdx();
    ParseTilingData(tilingGm);

    myOffset_ = blockIdx_ * tiling_.perCoreN;
    myCount_ = tiling_.perCoreN;
    if (blockIdx_ == GetBlockNum() - 1) {
        myCount_ += tiling_.remainder;
    }

    xGM_.SetGlobalBuffer((__gm__ float*)x, tiling_.totalN);
    yGM_.SetGlobalBuffer((__gm__ float*)y, tiling_.totalN);

    uint32_t bufSize = tiling_.tileSize * sizeof(float);
    pipe_->InitBuffer(inQueueX_, BUFFER_NUM, bufSize);
    pipe_->InitBuffer(inQueueY_, BUFFER_NUM, bufSize);
}

Init 做了三件事：

  1. 解析 Tiling 数据 — 从 GM 读取 Host 传来的切分参数
  2. 计算本核的数据范围 — myOffset_ 和 myCount_，最后一个核吸收余量
  3. 初始化 Buffer — 通过 TPipe 为 x 和 y 各分配一块 UB Buffer

8.2.4.3 SingleIteration：单次迭代

这是 swap 的核心逻辑。每次迭代处理一个 tile 大小的数据：

__aicore__ inline void SswapAIV::SingleIteration(uint32_t curOffset, uint32_t dataCount)
{
    uint32_t alignedCount = (dataCount / ELEMENTS_PER_BLOCK) * ELEMENTS_PER_BLOCK;
    uint32_t tailCount = dataCount - alignedCount;

    LocalTensor<float> xLocal = inQueueX_.AllocTensor<float>();
    LocalTensor<float> yLocal = inQueueY_.AllocTensor<float>();

    if (alignedCount > 0) {
        DataCopy(xLocal, xGM_[curOffset], alignedCount);
        DataCopy(yLocal, yGM_[curOffset], alignedCount);
    }

    if (tailCount > 0) {
        uint8_t paddingNum = ELEMENTS_PER_BLOCK - tailCount;
        DataCopyExtParams copyParams{1, static_cast<uint32_t>(tailCount * sizeof(float)), 0, 0, 0};
        DataCopyPadExtParams<float> padParams{true, 0, paddingNum, 0.0f};
        DataCopyPad(xLocal[alignedCount], xGM_[curOffset + alignedCount], copyParams, padParams);
        DataCopyPad(yLocal[alignedCount], yGM_[curOffset + alignedCount], copyParams, padParams);
    }

    inQueueX_.EnQue<float>(xLocal);
    inQueueY_.EnQue<float>(yLocal);

    LocalTensor<float> xIn = inQueueX_.DeQue<float>();
    LocalTensor<float> yIn = inQueueY_.DeQue<float>();

    if (alignedCount > 0) {
        DataCopy(yGM_[curOffset], xIn, alignedCount);
        DataCopy(xGM_[curOffset], yIn, alignedCount);
    }

    if (tailCount > 0) {
        DataCopyExtParams copyParams{1, static_cast<uint32_t>(tailCount * sizeof(float)), 0, 0, 0};
        DataCopyPad(yGM_[curOffset + alignedCount], xIn[alignedCount], copyParams);
        DataCopyPad(xGM_[curOffset + alignedCount], yIn[alignedCount], copyParams);
    }

    inQueueX_.FreeTensor(xIn);
    inQueueY_.FreeTensor(yIn);
}

代码分为四个阶段：

8.2.4.4 Process：主循环

__aicore__ inline void SswapAIV::Process()
{
    if (myCount_ == 0) {
        return;
    }

    uint32_t tileLoop = myCount_ / tiling_.tileSize;
    uint32_t tileTail = myCount_ % tiling_.tileSize;
    uint32_t curOffset = myOffset_;

    for (uint32_t i = 0; i < tileLoop; i++) {
        SingleIteration(curOffset, tiling_.tileSize);
        curOffset += tiling_.tileSize;
    }

    if (tileTail > 0) {
        SingleIteration(curOffset, tileTail);
    }
}

Process 将本核的数据按 tileSize 切分为多个迭代，每次调用 SingleIteration 处理一个 tile。最后处理尾部不足一个 tile 的数据。

8.2.4.5 Kernel 入口

extern "C" __global__ __aicore__ void sswap_kernel(
    GM_ADDR x, GM_ADDR y, GM_ADDR workSpace, GM_ADDR tilingGm)
{
    KERNEL_TASK_TYPE_DEFAULT(KERNEL_TYPE_AIV_ONLY);
    TPipe pipe;
    SswapAIV op;
    op.Init(x, y, tilingGm, &pipe);
    op.Process();
}

void sswap_kernel_do(
    GM_ADDR x, GM_ADDR y, GM_ADDR workSpace,
    GM_ADDR tilingGm, uint32_t numBlocks, void* stream)
{
    sswap_kernel<<<numBlocks, nullptr, stream>>>(x, y, nullptr, tilingGm);
}

注意两个关键点：

  • TPipe 在入口函数中创建，作为指针传给类的 Init 方法（而非作为类成员），这是 Ascend C 的规范要求
  • KERNEL_TYPE_AIV_ONLY 表示只使用 AIV（Vector 核），不使用 AIC（Cube 核），因为 swap 没有矩阵计算

8.2.5 Host 侧实现

Host 侧代码运行在 CPU 上，负责参数校验、Tiling 计算和 Kernel 启动。

8.2.5.1 参数校验

static aclblasStatus_t ValidateSswapParams(
    const float* x, const float* y, int incx, int incy)
{
    if (x == nullptr || y == nullptr) {
        OP_LOGE("aclblasSswap", "x/y must not be nullptr");
        return ACLBLAS_STATUS_INVALID_VALUE;
    }
    if (incx == 0 || incy == 0) {
        OP_LOGE("aclblasSswap", "incx and incy must not be zero");
        return ACLBLAS_STATUS_INVALID_VALUE;
    }
    if (incx != 1 || incy != 1) {
        OP_LOGE("aclblasSswap", "incx and incy must be 1, got incx=%d, incy=%d", incx, incy);
        return ACLBLAS_STATUS_INVALID_VALUE;
    }
    return ACLBLAS_STATUS_SUCCESS;
}

8.2.5.2 Tiling 计算

static SswapTilingData CalSswapTilingData(uint32_t n, uint32_t coreNum)
{
    SswapTilingData tiling;
    tiling.totalN = n;

    constexpr uint32_t alignUnit = ELEMENTS_PER_BLOCK;
    uint32_t rawPerCore = n / coreNum;
    tiling.perCoreN = (rawPerCore / alignUnit) * alignUnit;
    tiling.remainder = n - tiling.perCoreN * coreNum;

    constexpr uint32_t ubSize = 248 * 1024;
    constexpr uint32_t bufferCount = 2;
    uint32_t maxElements = ubSize / (bufferCount * sizeof(float));
    tiling.tileSize = (maxElements / alignUnit) * alignUnit;

    return tiling;
}

Tiling 计算的核心逻辑：

  1. 多核切分：将 n 个元素均分给 coreNum 个核，perCoreN 向下对齐到 8 的倍数
  2. 余量处理：remainder 由最后一个核吸收
  3. UB 切分：248KB UB 空间分给 x 和 y 两块 Buffer，每块 tileSize 个元素

8.2.5.3 Kernel 启动

aclblasStatus_t aclblasSswap(
    aclblasHandle_t handle, int n, float* x, int incx, float* y, int incy)
{
    if (n <= 0) {
        return ACLBLAS_STATUS_SUCCESS;
    }

    aclblasStatus_t st = ValidateSswapParams(x, y, incx, incy);
    if (st != ACLBLAS_STATUS_SUCCESS) {
        return st;
    }

    uint32_t aivCoreNum = GetVectorCoreCount();
    uint32_t totalN = static_cast<uint32_t>(n);
    uint32_t numBlocks = (totalN < aivCoreNum) ? totalN : aivCoreNum;

    SswapTilingData tiling = CalSswapTilingData(totalN, numBlocks);

    uint8_t* tilingDevice = nullptr;
    aclrtMalloc(reinterpret_cast<void**>(&tilingDevice),
                sizeof(SswapTilingData), ACL_MEM_MALLOC_HUGE_FIRST);
    aclrtMemcpy(tilingDevice, sizeof(SswapTilingData),
                &tiling, sizeof(SswapTilingData), ACL_MEMCPY_HOST_TO_DEVICE);

    sswap_kernel_do(
        reinterpret_cast<GM_ADDR>(x), reinterpret_cast<GM_ADDR>(y),
        nullptr, tilingDevice, numBlocks, useStream);

    aclrtFree(tilingDevice);
    return ACLBLAS_STATUS_SUCCESS;
}

8.2.7 关键设计决策

8.2.7.1 单 Buffer 的选择原因

双 Buffer（ping-pong）的目的是让搬运和计算重叠执行：当 Vector 单元在处理 ping Buffer 的数据时，MTE 同时向 pong Buffer 搬运下一批数据。

但 swap 算子没有 Vector 计算阶段。它的整个流程只有 MTE2（GM→UB）和 MTE3（UB→GM），没有中间的 Vector 计算可以与搬运重叠。因此双 Buffer 没有任何收益，反而浪费一半的 UB 空间。使用单 Buffer 可以让 tileSize 更大，减少迭代次数。

8.2.7.2 交叉写回的选择原因

一种直觉做法是：读入 x 和 y，在 UB 中交换，然后写回。但这需要额外的临时 Buffer 来暂存数据。交叉写回的技巧是：

  • 从 xLocal（UB 中存的 x 数据）写到 yGM（GM 中 y 的位置）
  • 从 yLocal（UB 中存的 y 数据）写到 xGM（GM 中 x 的位置）

这样不需要任何临时存储，直接在写回阶段完成交换。

Q&A

Q: 为什么 perCoreN 要对齐到 ELEMENTS_PER_BLOCK？

DataCopy 要求搬运的元素数按 32 字节对齐（对于 float 就是 8 个元素）。如果 perCoreN 不对齐，每个核的起始偏移就不是 32 字节对齐的，会导致 DataCopy 报错或数据错误。

Q: 如果 n 小于核数怎么办？

Host 侧会做 numBlocks = min(n, aivCoreNum) 的限制。当 n 很小时，只启动 n 个核，每个核处理 1 个元素。如果 n=0，直接返回成功，不启动 Kernel。

Q: TilingData 为什么要通过 aclrtMalloc + aclrtMemcpy 传递？

Kernel 运行在 Device 侧，只能访问 GM（全局内存）。TilingData 在 Host 侧计算完成后，必须拷贝到 Device 的 GM 上，Kernel 才能通过 GM_ADDR 参数读取。

Q: 为什么 swap 用直调而不用 aclnn 注册？

BLAS Level 1 算子（如 swap、scal、copy）接口简单，不需要 aclnn 框架提供的自动类型推导、Workspace 管理等高级功能。直调方式代码更简洁，启动开销更小。aclnn 注册更适合复杂算子（如 add、matmul），详见 8.3 add 算子开发。
