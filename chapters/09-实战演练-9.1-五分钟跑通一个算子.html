9.1 五分钟跑通一个算子

本章用最少的步骤让你在一台昇腾 NPU 上完成 “编译 → 运行 → 看到结果” 的完整流程。你不需要理解每一行代码——先跑起来，再慢慢学。

前提条件

在开始之前，请确认以下环境已就绪：

条件验证方式预期输出
CANN Toolkit 已安装ascend-smi 或 npu-smi info显示设备信息，含芯片型号
NPU 设备可用npu-smi info至少一块 NPU 状态为 OK
C++ 编译器g++ --versiongcc 7.3+ 或等效版本
Gitgit --version任意版本

如果 CANN 尚未安装，请参考 2.3 Ascend 软件栈 中的安装说明。

第一步：获取代码

ops-blas 是一个开源的 BLAS 算子工程仓库，包含了多个算子的完整实现。克隆到本地：

git clone https://gitcode.com/ascend/ops-blas.git
cd ops-blas

仓库的顶层目录结构如下：

ops-blas/
├── blas/              # 算子实现目录
│   ├── swap/          # swap 算子族
│   │   └── sswap/     # 单精度 swap
│   │       ├── arch22/
│   │       └── arch35/
│   ├── scal/
│   ├── copy/
│   └── ...
├── build.sh           # 一键编译脚本
└── CMakeLists.txt

第二步：编译 swap 算子

使用 build.sh 脚本一键编译。--ops 指定算子名，--soc 指定目标芯片：

bash build.sh --ops=swap --soc=ascend950

编译成功后，产物位于 build/out/ 目录：

build/out/
├── lib/
│   └── libascendc_blas.so    # 算子动态库
└── st/
    └── swap_test              # 测试可执行文件

第三步：运行测试

加上 --run 参数即可编译并自动运行 ST 测试：

bash build.sh --ops=swap --soc=ascend950 --run

测试框架会自动完成以下步骤：

  1. 在 Host 侧生成随机输入数据（向量 x 和 y）
  2. 用 CPU 计算 golden 结果（交换 x 和 y）
  3. 将数据拷贝到 NPU，调用算子执行
  4. 将 NPU 结果拷回 Host，与 golden 逐元素比对
  5. 输出 PASS / FAIL

预期输出：

全部用例显示 [ PASSED ] 即表示算子功能正确。

第四步：查看结果和日志

如果测试失败，可以查看详细日志来定位问题：

export ASCEND_GLOBAL_LOG_LEVEL=1
bash build.sh --ops=swap --soc=ascend950 --run

日志级别说明：

级别值内容
DEBUG0Tiling 参数、每个核的数据分配
INFO1Kernel 启动信息
WARNING2非致命异常
ERROR3错误信息（参数校验失败、内存分配失败等）

第五步：修改代码 → 重新编译 → 运行

开发算子的典型工作流是一个快速迭代循环：

试试做一个小修改来体验这个循环。打开 blas/swap/sswap/arch35/sswap_host.cpp，找到 OP_LOGI 那行，修改日志内容，然后重新编译运行：

bash build.sh --ops=swap --soc=ascend950 --run

你会发现编译速度很快——build.sh 使用了增量编译，只重新编译修改过的文件。

常见问题

Q: 编译报错找不到头文件 kernel_operator.h

这是因为 CANN Toolkit 的环境变量没有生效。请执行 source /usr/local/Ascend/ascend-toolkit/set_env.sh（路径根据你的安装位置调整），然后重新编译。

Q: 运行时报错 "no device found"

说明 NPU 设备不可用。先用 npu-smi info 检查设备状态。如果是容器环境，确认启动时已挂载设备（--device=/dev/davinci0）。

Q: --soc 参数应该填什么？

根据你的芯片型号选择：ascend910b 对应 A2（arch22），ascend950 对应 A5（arch35）。可以用 npu-smi info 查看芯片型号。

小结

恭喜你！你已经完成了第一个算子的编译和运行。整个流程可以概括为：

  1. 获取代码 — git clone 拉取算子工程
  2. 编译 — build.sh --ops=xxx --soc=yyy
  3. 运行 — 加上 --run 自动执行 ST 测试
  4. 迭代 — 修改代码 → 重新编译 → 运行，直到全部 PASS
