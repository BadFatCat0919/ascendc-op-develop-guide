4.3 swap 算子需求分析

在动手写代码之前，我们需要先理解 swap 算子的功能、参数和边界条件。这是算子开发的第一步——需求分析。

4.3.1 功能说明

swap 是 BLAS Level 1 标准中的算子，功能是交换两个向量的对应元素。执行 swap(x, y) 后，x 变成原来的 y，y 变成原来的 x。

BLAS Level 1 简介

BLAS（Basic Linear Algebra Subprograms）是线性代数的基础库，分为三个级别：

级别复杂度典型算子说明
Level 1O(n)swap, axpy, dot, nrm2向量-向量操作
Level 2O(n²)gemv, ger, trsv矩阵-向量操作
Level 3O(n³)gemm, trsm, syrk矩阵-矩阵操作

swap 属于 Level 1，是最简单的一类算子，非常适合作为入门练习。

数学定义

// BLAS 标准接口
void cblas_sswap(const int N, float *X, const int incX, float *Y, const int incY);

// 伪代码
for (int i = 0; i < N; i++) {
    float temp = X[i * incX];
    X[i * incX] = Y[i * incY];
    Y[i * incY] = temp;
}

功能示意

4.3.2 参数说明

swap 算子的参数非常简单，只有 5 个：

参数类型说明示例
Nint向量元素个数1000
Xfloat*第一个向量（输入/输出）[1.0, 2.0, 3.0, ...]
incXintX 的步长（通常为 1）1
Yfloat*第二个向量（输入/输出）[5.0, 6.0, 7.0, ...]
incYintY 的步长（通常为 1）1

步长（incX/incY）的含义

步长参数控制向量元素的访问间隔：

  • incX = 1：连续访问，X[0], X[1], X[2], ...（最常用）
  • incX = 2：每隔一个元素访问，X[0], X[2], X[4], ...
  • incX = -1：反向访问，X[N-1], X[N-2], ...

在本教程中，我们只实现 incX = incY = 1 的情况，这是最常见的场景。

4.3.3 边界条件

在实现算子时，需要处理以下边界条件：

1. N ≤ 0

如果 N 小于等于 0，算子应该直接返回，不做任何操作。

if (N <= 0) {
    return;  // 无需处理
}

2. N 很小

如果 N 很小（比如 N=10），使用多个 AI Core 反而会增加启动开销。需要设置一个最小值，确保每个 Core 至少处理一定数量的元素。

int32_t minElementsPerCore = 256;
int32_t coreCount = (N + minElementsPerCore - 1) / minElementsPerCore;
if (coreCount > maxCoreCount) {
    coreCount = maxCoreCount;
}

3. N 不能被 Core 数量整除

如果 N 不能被 Core 数量整除，最后一个 Core 需要处理余数部分。

int32_t perCoreN = N / coreCount;
int32_t remainder = N % coreCount;
// 最后一个 Core 处理 perCoreN + remainder 个元素

4. 空指针

如果 X 或 Y 是空指针，应该报错并返回。

if (x == nullptr || y == nullptr) {
    std::cerr << "Error: x and y cannot be null" << std::endl;
    return -1;
}

4.3.4 文件结构

swap 算子的代码位于 blas/swap/arch35/ 目录下，包含三个核心文件：

blas/swap/arch35/

├── swap_kernel.cpp        # Kernel 侧实现（~100 行）
└── swap_tiling_data.h     # Tiling 数据结构（~20 行）

Q&A：常见问题

Q1：为什么 swap 是原地操作（in-place）？

A：swap 的输入和输出使用相同的内存地址。执行后，X 和 Y 的内容互相交换，不需要额外的输出缓冲区。这种设计节省了内存空间。

Q2：swap 和 copy 有什么区别？

A：copy 是把 X 的内容复制到 Y（Y = X），X 保持不变。swap 是交换 X 和 Y 的内容，两者都会改变。

Q3：为什么 BLAS 标准使用列主序（Column-Major）？

A：BLAS 起源于 Fortran，Fortran 默认使用列主序存储矩阵。对于向量操作（Level 1），列主序和行主序没有区别，因为向量是一维的。
