5.5 同步与队列 API 详解

同步与队列 API 是 Ascend C 多阶段流水线编程的核心基础设施。NPU 内部有多个硬件引擎（MTE、Vector、Cube）并行工作，同步 API 协调它们的执行顺序，队列 API 管理它们之间的数据传递。本节详细介绍 SetFlag/WaitFlag、TQue/TQueBind、TPipe 和 EnQue/DeQue 的用法。

5.5.1 SetFlag / WaitFlag

SetFlag 和 WaitFlag 是 NPU 内部 Pipe 之间的同步原语。每个 Pipe（MTE2、MTE3、Vector、Cube、Scalar）是独立的硬件执行单元，SetFlag/WaitFlag 确保它们按正确的顺序访问共享数据。

5.5.1.1 函数签名

template <typename T>
__aicore__ inline void SetFlag(Pipe pipe, const T& flag);

template <typename T>
__aicore__ inline void WaitFlag(Pipe pipe, const T& flag);
参数类型说明
pipePipe 枚举目标 Pipe：PIPE_MTE2, PIPE_MTE3, PIPE_V, PIPE_CUBE, PIPE_S
flagflag 对象同步标志位对象，用于标识特定的同步点

5.5.1.2 Pipe 对

SetFlag/WaitFlag 总是成对使用，形成 Pipe 对（Pipe Pair）。常见的 Pipe 对包括：

生产者 Pipe消费者 Pipe典型场景
MTE2（搬入）Vector（计算）数据搬到 UB 后，通知 Vector 开始计算
Vector（计算）MTE3（搬出）计算完成后，通知 MTE3 搬出结果
MTE2（搬入）Cube（计算）数据搬到 L0A/L0B 后，通知 Cube 开始计算
Cube（计算）Vector（后处理）Cube 输出后，通知 Vector 做 FixPipe

5.5.1.3 flag 的生命周期

每个 flag 对象维护一个计数器。SetFlag 递增计数器，WaitFlag 等待计数器到达指定值后递减。通过计数器机制，可以支持多对多的同步关系。

PipeBarrier<PIPE_ALL> flagMte2V;
PipeBarrier<PIPE_ALL> flagVMte3;

SetFlag(PIPE_MTE2, flagMte2V);
WaitFlag(PIPE_V, flagMte2V);

SetFlag(PIPE_V, flagVMte3);
WaitFlag(PIPE_MTE3, flagVMte3);

5.5.2 BufferID 机制（A5 特有）

A5 架构引入了 BufferID 机制，用于更精细的同步控制。每个 Buffer 有一个唯一的 BufferID，SetFlag/WaitFlag 可以针对特定的 BufferID 进行同步，而不是全局同步。

BufferID 机制的优势在于：当使用多缓冲（如双缓冲或三缓冲）时，可以精确地同步到某个特定的 Buffer，避免不必要的等待。例如，Vector 计算 Buffer 0 时，MTE2 可以同时搬运 Buffer 1 的数据，两者互不干扰。

5.5.3 TQue / TQueBind

TQue 和 TQueBind 是 Ascend C 提供的队列抽象，用于管理 Buffer 池和实现生产者-消费者模式。

5.5.3.1 TQue

template <TPosition pos, uint8_t bufferNum>
class TQue;
模板参数说明
pos队列位置：VECIN, VECOUT, MTE2, MTE3
bufferNumBuffer 个数（队列深度），通常为 1 或 2

5.5.3.2 队列位置说明

位置说明典型用途
VECINVector 输入队列MTE2 搬入的数据暂存处，Vector 从此取数据
VECOUTVector 输出队列Vector 计算结果暂存处，MTE3 从此取数据搬出
MTE2MTE2 搬入队列MTE2 阶段的 Buffer 管理
MTE3MTE3 搬出队列MTE3 阶段的 Buffer 管理

5.5.3.3 TQueBind

template <TPosition srcPos, TPosition dstPos, uint8_t bufferNum>
class TQueBind;

TQueBind 连接两个 Pipe 阶段，自动管理跨阶段的 Buffer 流转。例如 TQueBind<VECIN, VECOUT, 2> 创建一个从 VECIN 到 VECOUT 的绑定队列，深度为 2。

5.5.4 TPipe 与 InitBuffer

5.5.4.1 TPipe

class TPipe;

TPipe 是整个算子 Kernel 的顶层容器，管理所有队列的 Buffer 分配和生命周期。每个 Kernel 函数中通常只有一个 TPipe 实例。

5.5.4.2 InitBuffer

template <TPosition pos, uint8_t bufferNum>
void InitBuffer(TQue<pos, bufferNum>& que, uint8_t bufferNum, size_t bufferSize);
参数说明
que要初始化的队列对象
bufferNumBuffer 个数（与 TQue 模板参数一致）
bufferSize每个 Buffer 的大小（字节）
 
TPipe pipe;
TQue<TPosition::VECIN, 2> queIn;
TQue<TPosition::VECOUT, 1> queOut;

pipe.InitBuffer(queIn, 2, 2048);
pipe.InitBuffer(queOut, 1, 2048);

5.5.5 EnQue / DeQue

EnQue 和 DeQue 实现生产者-消费者模式的 Buffer 传递。

5.5.5.1 EnQue（入队）

que.EnQue(tensor);

生产者将填充好的 Tensor 放入队列，通知消费者数据已就绪。

5.5.5.2 DeQue（出队）

que.DeQue(tensor);

消费者从队列中取出一个 Tensor，获取数据进行处理。如果队列为空，DeQue 会阻塞等待。

5.5.5.3 AllocTensor / FreeTensor

que.AllocTensor(tensor);
que.FreeTensor(tensor);

AllocTensor 从队列的 Buffer 池中分配一个 Buffer，FreeTensor 归还。通常在 EnQue 之前 AllocTensor，在 DeQue 之后 FreeTensor。

5.5.5.4 完整流水线示例

TPipe pipe;
TQue<TPosition::VECIN, 2> queIn;
TQue<TPosition::VECOUT, 1> queOut;
pipe.InitBuffer(queIn, 2, tileSize * sizeof(T));
pipe.InitBuffer(queOut, 1, tileSize * sizeof(T));

for (int i = 0; i < tileNum; i++) {
    LocalTensor<T> tmpIn;
    queIn.AllocTensor(tmpIn);
    DataCopy(tmpIn, gmIn + i * tileSize, tileSize);
    queIn.EnQue(tmpIn);

    LocalTensor<T> tmpOut;
    queOut.AllocTensor(tmpOut);

    LocalTensor<T> inData;
    queIn.DeQue(inData);
    Adds(tmpOut, inData, 1.0f, tileSize, repeat, rp);
    queIn.FreeTensor(inData);
    queOut.EnQue(tmpOut);

    LocalTensor<T> outData;
    queOut.DeQue(outData);
    DataCopy(gmOut + i * tileSize, outData, tileSize);
    queOut.FreeTensor(outData);
}

Q&A

Q：SetFlag/WaitFlag 和 ACL 的 Stream/Event 有什么区别？

A：它们工作在不同层级。SetFlag/WaitFlag 是 NPU 内部 Pipe 之间的同步（Kernel 侧），协调 MTE、Vector、Cube 等硬件引擎的执行顺序。Stream/Event 是 Host 侧的同步机制（ACL 层），用于管理多个 Kernel 之间的执行顺序和依赖关系。

Q：什么时候用 TQue，什么时候用 TQueBind？

A：TQue 用于单一 Pipe 阶段内的 Buffer 管理，你需要手动调用 EnQue/DeQue 来传递数据。TQueBind 连接两个 Pipe 阶段，自动管理跨阶段的 Buffer 流转，代码更简洁。简单算子用 TQue 即可，复杂多阶段流水线用 TQueBind 更方便。

Q：队列深度（bufferNum）设为 1 和 2 有什么区别？

A：深度为 1 表示单缓冲——搬运和计算不能重叠，必须等搬运完才能计算。深度为 2 表示双缓冲——搬运和计算可以重叠执行（一个 Buffer 在计算时，另一个在搬运），性能更好但占用更多 UB 空间。选择取决于 UB 容量和性能需求。

Q：DeQue 会阻塞吗？

A：会。如果队列为空（没有可消费的 Buffer），DeQue 会阻塞当前 Pipe 直到有数据入队。这是生产者-消费者模式的天然行为：消费者必须等生产者把数据准备好。EnQue 也可能阻塞——如果队列已满（所有 Buffer 都在被消费），生产者必须等消费者释放 Buffer。
