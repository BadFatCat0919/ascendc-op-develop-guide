4.1 AscendC 常用 API 概览

Ascend C 提供了一套完整的 C++ API，用于在昇腾 NPU 上开发高性能算子。这些 API 按功能可分为八大类别：数据搬运、逐元素计算、归约、类型转换、矩阵计算、同步、队列和流水线。本节对每类 API 做概览介绍，后续章节将逐类展开详解。

4.1.1 数据搬运 API

数据搬运 API 负责在 NPU 的不同存储层级之间移动数据。它们驱动 MTE（Memory Transfer Engine）引擎执行，与计算单元并行工作。

API签名说明典型用法
DataCopyDataCopy(dst, src, count)在 GM 与 UB 之间拷贝 count 个元素DataCopy(ubOut, gmIn, 128);
DataCopyPadDataCopyPad(dst, src, padParams)带 padding 的数据拷贝，用于补齐对齐DataCopyPad(ubOut, gmIn, params);
CopyInCopyIn(dst, src, count)Cube 场景：GM → L0A/L0BCopyIn(l0a, gmA, 256);
CopyOutCopyOut(dst, src, count)Cube 场景：L0C → GMCopyOut(gmC, l0c, 128);

4.1.2 逐元素计算 API

逐元素计算 API 对输入 Tensor 的每个元素独立执行相同运算，由 Vector 单元执行。所有 API 都支持 repeat/mask 机制来处理大数据量。

API签名说明典型用法
AddAdd(dst, src0, src1, len, repeat, rp)逐元素加Add(c, a, b, 128, 1, rp);
SubSub(dst, src0, src1, len, repeat, rp)逐元素减Sub(c, a, b, 128, 1, rp);
MulMul(dst, src0, src1, len, repeat, rp)逐元素乘Mul(c, a, b, 128, 1, rp);
DivDiv(dst, src0, src1, len, repeat, rp)逐元素除Div(c, a, b, 128, 1, rp);
MaxMax(dst, src0, src1, len, repeat, rp)逐元素取大Max(c, a, b, 128, 1, rp);
MinMin(dst, src0, src1, len, repeat, rp)逐元素取小Min(c, a, b, 128, 1, rp);
AbsAbs(dst, src, len, repeat, rp)逐元素绝对值Abs(b, a, 128, 1, rp);
ExpExp(dst, src, len, repeat, rp)逐元素指数Exp(b, a, 128, 1, rp);
LnLn(dst, src, len, repeat, rp)逐元素自然对数Ln(b, a, 128, 1, rp);
SqrtSqrt(dst, src, len, repeat, rp)逐元素平方根Sqrt(b, a, 128, 1, rp);
ReciprocalReciprocal(dst, src, len, repeat, rp)逐元素倒数Reciprocal(b, a, 128, 1, rp);
AddsAdds(dst, src, scalar, len, repeat, rp)向量 + 标量Adds(b, a, 1.0f, 128, 1, rp);
MulsMuls(dst, src, scalar, len, repeat, rp)向量 × 标量Muls(b, a, 0.5f, 128, 1, rp);

4.1.3 归约 API

归约 API 沿指定维度聚合数据，将多个元素压缩为更少的结果。常用于求和、求最值等场景。

API签名说明典型用法
ReduceSumReduceSum(dst, src, len)求所有元素之和ReduceSum(out, in, 1024);
ReduceMaxReduceMax(dst, src, len)求所有元素中的最大值ReduceMax(out, in, 1024);
ReduceMinReduceMin(dst, src, len)求所有元素中的最小值ReduceMin(out, in, 1024);

4.1.4 类型转换 API

类型转换 API 在不同数据类型之间转换，由 Vector 单元执行。精度转换时需注意信息损失。

API签名说明典型用法
CastCast(dst, src, mode, len, repeat, rp)数据类型转换（FP32↔FP16↔BF16↔INT8 等）Cast(dstFp16, srcFp32, 0, 128, 1, rp);

4.1.5 矩阵计算 API

矩阵计算 API 驱动 Cube 单元执行矩阵乘法及相关后处理操作。

API签名说明典型用法
MatmulMatmul(dst, srcA, srcB, shape)矩阵乘法 C = A × BMatmul(l0c, l0a, l0b, {16,16,16});
MatmulLeakyReluMatmulLeakyRelu(dst, srcA, srcB, shape, alpha)矩阵乘法 + LeakyReLU 融合MatmulLeakyRelu(l0c, l0a, l0b, shape, 0.01f);
FixPipeFixPipe(dst, src, len, repeat, rp)Cube 输出后处理（量化、精度转换等）FixPipe(ubOut, l0c, 128, 1, rp);

4.1.6 同步 API

同步 API 用于协调 NPU 内部多个 Pipe（流水线阶段）之间的执行顺序，防止数据竞争。

API签名说明典型用法
SetFlagSetFlag(pipe, flagId)设置同步标志位，通知下游 Pipe 数据已就绪SetFlag(PIPE_MTE2, flag);
WaitFlagWaitFlag(pipe, flagId)等待同步标志位，阻塞直到上游 Pipe 完成WaitFlag(PIPE_V, flag);

4.1.7 队列 API

队列 API 提供生产者-消费者模式的 Buffer 管理，是多阶段流水线编程的基础设施。

API签名说明典型用法
TQueTQue<T, stage> que声明一个队列对象TQue<TPosition::VECIN, 1> queIn;
TQueBindTQueBind<src, dst, depth> que声明绑定队列，连接两个 Pipe 阶段TQueBind<VECIN, VECOUT, 2> queBind;
EnQueque.EnQue(tensor)将 Tensor 入队（生产者）queIn.EnQue(tmpIn);
DeQueque.DeQue(tensor)从队列取出 Tensor（消费者）queIn.DeQue(tmpIn);
AllocTensorque.AllocTensor(tensor)从队列分配一个 Tensor BufferqueIn.AllocTensor(tmpIn);
FreeTensorque.FreeTensor(tensor)将 Tensor Buffer 归还队列queIn.FreeTensor(tmpIn);

4.1.8 流水线 API

流水线 API 管理 TPipe 对象的生命周期和 Buffer 初始化，是整个算子 Kernel 的顶层容器。

API签名说明典型用法
TPipeTPipe pipe声明流水线对象TPipe pipe;
InitBufferpipe.InitBuffer(que, depth, size)为队列分配 Buffer 空间pipe.InitBuffer(queIn, 2, 1024);

Q&A

Q：DataCopy 和 CopyIn/CopyOut 有什么区别？

A：DataCopy 用于 Vector 场景，在 GM 和 UB 之间搬运数据。CopyIn/CopyOut 用于 Cube 场景，在 GM 和 L0A/L0B/L0C 之间搬运数据。选择哪个取决于你的算子使用 Vector 单元还是 Cube 单元。

Q：Adds/Muls 和 Add/Mul 的区别是什么？

A：Add/Mul 是两个向量之间的运算（C = A + B），Adds/Muls 是向量与标量的运算（C = A + scalar）。Adds/Muls 在 BatchNorm 缩放、量化 scale 等场景中非常常用。

Q：什么时候用 TQue，什么时候用 TQueBind？

A：TQue 用于单一 Pipe 阶段内的 Buffer 管理（如 MTE2 阶段的输入队列）。TQueBind 用于连接两个不同 Pipe 阶段（如 VECIN → VECOUT），自动管理跨阶段的数据流转。简单场景用 TQue 即可，多阶段流水线用 TQueBind 更方便。
