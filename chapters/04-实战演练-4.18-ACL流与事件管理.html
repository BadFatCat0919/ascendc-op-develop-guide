4.14 ACL 流与事件管理

Stream（流）和 Event（事件）是 ACL 中控制异步执行和同步的核心机制。理解它们对于正确使用算子至关重要。

流管理

Stream（流）是 NPU 上的异步执行队列。同一个 Stream 内的操作按提交顺序执行，不同 Stream 之间的操作可以并行。

aclrtCreateStream

aclError aclrtCreateStream(aclrtStream *stream);

作用：创建一个新的执行流。创建后可以用它来启动 Kernel 和执行内存拷贝。

aclrtDestroyStream

aclError aclrtDestroyStream(aclrtStream stream);

作用：销毁指定执行流，释放相关资源。

aclrtSynchronizeStream

aclError aclrtSynchronizeStream(aclrtStream stream);

作用：阻塞等待指定 Stream 中的所有操作完成。这是最常用的同步方式——Kernel 启动是异步的，调用后不会等 Kernel 跑完就返回，必须用 SynchronizeStream 来等待。

aclrtGetCurrentContext

aclError aclrtGetCurrentContext(aclrtContext *context);

作用：获取当前线程的 Context。Context 是设备上的运行环境，一个设备可以有多个 Context。

// 流管理的典型用法
aclrtStream stream = nullptr;
aclrtCreateStream(&stream);

// 使用 stream 启动 Kernel（异步，立即返回）
my_kernel<<<coreNum, nullptr, stream>>>(devA, devB, devC, tilingGm);

// 等待 Kernel 执行完成
aclrtSynchronizeStream(stream);

// 此时才能安全地拷贝结果
aclrtMemcpy(hostOutput, size, devOutput, size, ACL_MEMCPY_DEVICE_TO_HOST);

// 销毁流
aclrtDestroyStream(stream);

常见错误：启动 Kernel 后没有调用 aclrtSynchronizeStream 就直接拷贝结果。由于 Kernel 是异步执行的，此时结果还没算完，拷贝回来的数据是未初始化的垃圾值。

事件管理

Event（事件）用于流间同步——当一个 Stream 的操作依赖另一个 Stream 的结果时，用 Event 来建立依赖关系。

aclrtCreateEvent

aclError aclrtCreateEvent(aclrtEvent *event);

作用：创建一个事件对象。

aclrtRecordEvent

aclError aclrtRecordEvent(aclrtEvent event, aclrtStream stream);

作用：在指定 Stream 中记录一个事件。当 Stream 执行到该事件时，事件被标记为"已完成"。

aclrtStreamWaitEvent

aclError aclrtStreamWaitEvent(aclrtStream stream, aclrtEvent event);

作用：让指定 Stream 等待某个事件完成后才继续执行后续操作。

aclrtSynchronizeEvent

aclError aclrtSynchronizeEvent(aclrtEvent event);

作用：阻塞等待指定事件完成。

aclrtDestroyEvent

aclError aclrtDestroyEvent(aclrtEvent event);

作用：销毁事件对象。

// 流间同步示例：Stream B 依赖 Stream A 的结果
aclrtEvent event = nullptr;
aclrtCreateEvent(&event);

// Stream A 执行计算
kernelA<<<coreNum, nullptr, streamA>>>(...);
aclrtRecordEvent(event, streamA);  // 在 Stream A 中记录事件

// Stream B 等待事件完成后再执行
aclrtStreamWaitEvent(streamB, event);
kernelB<<<coreNum, nullptr, streamB>>>(...);

// 清理
aclrtSynchronizeStream(streamA);
aclrtSynchronizeStream(streamB);
aclrtDestroyEvent(event);

单流 vs 多流

在大多数算子开发场景中，使用单流就足够了。多流主要用于需要并行执行多个独立算子的场景。

场景推荐方式说明
单个算子执行单流创建一个 Stream，启动 Kernel，同步，取结果
多个独立算子多流每个算子一个 Stream，可并行执行提升吞吐
有依赖的算子链单流 + Event用 Event 建立依赖，保证执行顺序

Q&A

Q：Kernel 启动后为什么是异步的？不能改成同步吗？

A：异步执行是 NPU 的设计特性——Host 提交任务后立即返回，不等待 Device 执行完成。这样可以避免 CPU 空等，提升整体效率。ACL 没有提供"同步启动 Kernel"的接口，必须通过 aclrtSynchronizeStream 显式等待。

Q：aclrtMemcpy 是同步的还是异步的？

A：aclrtMemcpy 默认是同步的（阻塞等待拷贝完成）。如果需要异步拷贝，使用 aclrtMemcpyAsync，它需要传入 Stream 参数，拷贝操作会被放入 Stream 的异步队列中。

Q：如果只用一个 Stream，还需要 Event 吗？

A：不需要。同一个 Stream 内的操作天然按提交顺序执行，不需要 Event 来同步。Event 只在需要跨 Stream 同步时才有用。
