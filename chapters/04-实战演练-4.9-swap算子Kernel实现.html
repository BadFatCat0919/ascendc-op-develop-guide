4.5 swap 算子 Kernel 实现

Kernel 是运行在 NPU AI Core 上的核函数，负责实际的数据搬运。本节将逐函数讲解 swap 算子的 Kernel 实现。

4.5.1 完整代码概览

// swap_kernel.cpp
#include "kernel_operator.h"
#include "swap_tiling_data.h"

using namespace AscendC;

// Ascend C Kernel 类
class SwapKernel {
public:
    __aicore__ inline void Init(GM_ADDR x, GM_ADDR y, const SwapTilingData &tiling);
    __aicore__ inline void Process();
    __aicore__ inline void SingleIteration(int32_t offset, int32_t size);

private:
    GlobalTensor<float> xGm;    // X 向量的全局内存指针
    GlobalTensor<float> yGm;    // Y 向量的全局内存指针
    LocalTensor<float> xBuf;    // X 的片上缓存
    LocalTensor<float> yBuf;    // Y 的片上缓存
    int32_t myN;                // 当前 Core 需要处理的元素个数
    int32_t tileSize;           // 每次搬运的元素个数
};

// Kernel 入口函数（C 接口）
extern "C" __global__ __aicore__ void swap_kernel_entry(
    GM_ADDR x, GM_ADDR y, GM_ADDR tiling_ptr);

4.5.2 Init 函数

Init 函数负责初始化当前 AI Core 的工作参数：确定自己负责的数据范围，分配 UB Buffer。

__aicore__ inline void Init(GM_ADDR x, GM_ADDR y, const SwapTilingData &tiling) {
    // 1. 获取当前 Core 的索引
    int32_t coreIdx = GetBlockIdx();
    
    // 2. 计算当前 Core 负责的数据范围
    int32_t offset = coreIdx * tiling.perCoreN;
    int32_t myN = tiling.perCoreN;
    
    // 3. 最后一个 Core 需要处理余数
    if (coreIdx == tiling.coreCount - 1) {
        myN += tiling.remainder;
    }
    
    // 4. 保存当前 Core 的参数
    this->myN = myN;
    this->tileSize = tiling.tileSize;
    
    // 5. 设置 GM Buffer（指向全局内存）
    xGm.SetGlobalBuffer(reinterpret_cast<float*>(x + offset * sizeof(float)));
    yGm.SetGlobalBuffer(reinterpret_cast<float*>(y + offset * sizeof(float)));
    
    // 6. 分配 UB Buffer（片上缓存）
    xBuf.Allocate(this->tileSize);
    yBuf.Allocate(this->tileSize);
}

Init 函数执行流程

4.5.3 Process 函数

Process 函数是主循环，负责把所有数据分块处理完毕。

__aicore__ inline void Process() {
    int32_t processed = 0;
    
    // 循环处理，每次处理 tileSize 个元素
    while (processed < myN) {
        int32_t currentSize = (myN - processed < tileSize) 
                              ? (myN - processed) 
                              : tileSize;
        SingleIteration(processed, currentSize);
        processed += currentSize;
    }
}

循环逻辑

Process 函数的核心是一个 while 循环：

  • 每次调用 SingleIteration 处理一个数据块
  • 最后一块可能不足 tileSize，需要特殊处理
  • 循环直到所有 myN 个元素都处理完毕

4.5.4 SingleIteration 函数

SingleIteration 是单次迭代的核心逻辑：搬入数据，交叉写回。

__aicore__ inline void SingleIteration(int32_t offset, int32_t size) {
    // 1. 从 GM 搬入到 UB（MTE2：Memory Transfer Engine 2，GM→UB）
    DataCopy(xBuf, xGm[offset], size);
    DataCopy(yBuf, yGm[offset], size);
    
    // 2. 交叉写回：x 写回 y，y 写回 x（MTE3：UB→GM）
    DataCopy(xGm[offset], yBuf, size);  // y 的数据写到 x
    DataCopy(yGm[offset], xBuf, size);  // x 的数据写到 y
}

4.5.5 交叉写回技巧

swap 的核心逻辑是"交换"。在 NPU 上，我们用"交叉写回"技巧来高效实现。

传统做法（需要额外 Buffer）

// 传统做法：需要 4 个 Buffer
LocalTensor<float> xBuf, yBuf, tempX, tempY;

// 1. 搬入
DataCopy(xBuf, xGm[offset], size);
DataCopy(yBuf, yGm[offset], size);

// 2. 交换（需要额外 Buffer）
DataCopy(tempX, xBuf, size);
DataCopy(tempY, yBuf, size);

// 3. 写回
DataCopy(xGm[offset], tempY, size);  // x = 原来的 y
DataCopy(yGm[offset], tempX, size);  // y = 原来的 x

交叉写回（只需要 2 个 Buffer）

// 交叉写回：只需要 2 个 Buffer
LocalTensor<float> xBuf, yBuf;

// 1. 搬入
DataCopy(xBuf, xGm[offset], size);  // xBuf = x 的数据
DataCopy(yBuf, yGm[offset], size);  // yBuf = y 的数据

// 2. 交叉写回（关键！）
DataCopy(xGm[offset], yBuf, size);  // 把 y 的数据写到 x 的位置
DataCopy(yGm[offset], xBuf, size);  // 把 x 的数据写到 y 的位置

数据流可视化

性能对比

方案Buffer 数量UB 占用DataCopy 次数性能
传统做法4 个高（需要额外 temp Buffer）6 次较慢
交叉写回2 个低（只需要 xBuf 和 yBuf）4 次更快

交叉写回的核心思想是：不需要在 UB 内做交换，而是通过写回时的"交叉"来实现交换。xBuf 里存的是 x 的数据，但我们把它写到 y 的位置；yBuf 里存的是 y 的数据，但我们把它写到 x 的位置。

4.5.6 Kernel 入口函数

入口函数是 Kernel 的"大门"——接收 GM 参数，解析 TilingData，调用 Kernel 类。

extern "C" __global__ __aicore__ void swap_kernel_entry(
    GM_ADDR x, 
    GM_ADDR y, 
    GM_ADDR tiling_ptr) 
{
    // 从 GM 读取 Tiling 数据
    SwapTilingData tiling;
    GM_ADDR tilingAddr = tiling_ptr;
    for (int i = 0; i < sizeof(SwapTilingData) / sizeof(int32_t); i++) {
        reinterpret_cast<int32_t*>(&tiling)[i] = 
            reinterpret_cast<int32_t*>(tilingAddr)[i];
    }
    
    // 创建 Kernel 实例并执行
    SwapKernel kernel;
    kernel.Init(x, y, tiling);
    kernel.Process();
}

4.5.7 为什么用单 Buffer 而不是双 Buffer？

双 Buffer 的价值是让"搬运"和"计算"在不同 Buffer 上同时执行。但 swap 没有计算阶段——它只有搬运。搬运和搬运之间无法重叠（MTE2 和 MTE3 是独立的，但同一方向的搬运不能重叠），所以双 Buffer 对 swap 没有额外收益，反而浪费一倍 UB 空间。

Q&A：常见问题

Q1：为什么 Kernel 入口函数要用 extern "C"？

A：extern "C" 禁止 C++ 名称修饰（name mangling），确保 Host 侧能通过固定名称找到 Kernel 入口函数。如果不加，编译器会把函数名改成类似 _Z17swap_kernel_entryPvS_S_ 的修饰名。

Q2：DataCopy 的第三个参数是字节数还是元素数？

A：是元素数，不是字节数。Ascend C 的 DataCopy API 会根据 Tensor 的数据类型自动计算字节数。例如 DataCopy(xBuf, xGm[0], 256) 搬运 256 个 float 元素（1024 字节）。

Q3：为什么 SingleIteration 中两次 DataCopy 不需要同步？

A：因为 swap 的 DataCopy 操作是顺序执行的：先搬入 xBuf 和 yBuf，再交叉写回。搬入完成后才能写回，所以不需要显式同步。但在更复杂的算子中（如 add），计算和搬运可能并行，需要使用 PipeBarrier 同步。
