4.9 add 算子 Tiling 设计

4.9.1 TilingData 结构

TilingData 是 Host 和 Kernel 之间的通信协议——Host 侧计算好切分策略，填入 TilingData 结构体，Kernel 侧从 GM 中读取并解析它，据此决定自己该处理哪段数据。

对于 add 算子，TilingData 需要传递以下信息：

// add_tiling_data.h
#ifndef ADD_TILING_DATA_H
#define ADD_TILING_DATA_H

#include <cstdint>

struct AddTilingData {
    uint32_t totalLength;    // 向量总元素数
    uint32_t tileNum;        // 使用的核数
    uint32_t tileSize;       // 每个核处理的元素数
    uint32_t alignSize;      // 对齐后的元素数（32B 对齐）
};

#endif // ADD_TILING_DATA_H

各字段含义

字段类型含义计算方式
totalLengthuint32_t输入向量的总长度 N从输入 shape 获取
tileNumuint32_t实际使用的 AI Core 数量min(coreNum, totalLength)
tileSizeuint32_t每个核处理的元素数ceil(totalLength / tileNum)
alignSizeuint32_ttileSize 向上对齐到 32B 边界的元素数向上取整到 32B 倍数

4.9.2 切分策略

add 算子的切分策略是均匀切分：将长度为 N 的向量平均分配给多个 AI Core，每个核处理一段连续的数据。

切分计算步骤

  1. 获取总长度：从输入 tensor 的 shape 中获取元素总数 N
  2. 确定核数：查询硬件可用核数，取 min(coreNum, N)
  3. 计算 tileSize：ceil(N / tileNum)，每个核处理的数据量
  4. 计算 alignSize：tileSize 向上对齐到 32B 边界（DataCopy 要求）

对齐计算详解

DataCopy API 要求搬运长度必须是 32B 对齐的。对于 float（4 字节），32B 对齐意味着每次至少搬 8 个元素（32 / 4 = 8）。

// 对齐计算公式
uint32_t alignSize = ((tileSize * sizeof(float) + 31) / 32) * (32 / sizeof(float));

// 示例：tileSize = 250, sizeof(float) = 4
// alignSize = ((250 * 4 + 31) / 32) * (32 / 4)
//           = ((1000 + 31) / 32) * 8
//           = (1031 / 32) * 8
//           = 32 * 8
//           = 256

注意：alignSize 用于 DataCopy 搬入，processLength（实际数据长度）用于 Vector 计算和搬出。两者不同是 add 算子中最容易踩的坑之一。

4.9.3 Tiling 函数实现

Tiling 函数在 aclnn 框架的上下文中执行，通过 context 获取硬件信息和输入 shape：

// add_host.cpp（Tiling 函数部分）
#include "add_tiling_data.h"
#include "tiling/tiling_api.h"

static void AddTilingFunc(const aclnnTensorDesc* inputADesc,
                           const aclnnTensorDesc* inputBDesc,
                           const aclnnTensorDesc* outputCDesc,
                           AddTilingData* tilingData)
{
    // 1. 获取输入长度
    uint32_t totalLength = inputADesc->numElements;

    // 2. 获取可用核数
    uint32_t coreNum = 0;
    aclrtGetCoreNum(&coreNum);

    // 3. 计算每个核处理的元素数（向上取整）
    uint32_t tileSize = (totalLength + coreNum - 1) / coreNum;

    // 4. 32B 对齐
    uint32_t alignSize = ((tileSize * sizeof(float) + 31) / 32)
                         * (32 / sizeof(float));

    // 5. 填充 TilingData
    tilingData->totalLength = totalLength;
    tilingData->tileNum = coreNum;
    tilingData->tileSize = tileSize;
    tilingData->alignSize = alignSize;
}

Q&A

Q：为什么 tileNum 不能超过硬件核数？

A：每个 AI Core 是独立的计算单元，tileNum 超过物理核数会导致多个 tile 排队等待，反而降低效率。通常 tileNum = min(可用核数, totalLength)，当数据量很小时，不需要启动所有核。

Q：如果 totalLength 不能被 tileNum 整除怎么办？

A：最后一个核处理的数据量会少于 tileSize。Kernel 侧需要检查 offset + tileSize > totalLength，如果是则截断为 totalLength - offset。这就是 processLength 的作用。

Q：alignSize 一定大于等于 tileSize 吗？

A：是的。alignSize 是 tileSize 向上对齐到 32B 边界的结果，所以 alignSize ≥ tileSize。多出的对齐部分是无效数据，DataCopy 会搬运但不会参与计算。
