3.3 Ascend 软件栈

3.3.1 CANN 软件栈总览

CANN（Compute Architecture for Neural Networks）是昇腾 NPU 的软件栈，地位类似于 NVIDIA 的 CUDA。它负责把上层框架（PyTorch、MindSpore）的计算请求翻译成 NPU 能理解的指令，并管理硬件资源（AI Core、内存、数据搬运引擎等）。

把 CANN 想象成一家连锁奶茶品牌的管理系统：PyTorch/MindSpore 是前台 POS 机（接单），CANN 是后台调度中心（拆单、排产、品控），NPU 是厨房（实际制作）。你在 POS 机上点了一杯"满杯百香果"（调用 torch.add），调度中心把它拆成具体的工序（算子），安排到空闲的工位（AI Core）上执行，最后把成品送回前台。

上图展示了 CANN 软件栈的层次结构。作为算子开发者，你主要打交道的是"算子代码仓"和"CANN 编译工具链"这两层。算子代码用 Ascend C 编写，通过 ascendc 编译器编译为 NPU 可执行的二进制，再通过 ACL Runtime 加载到设备上运行。

3.3.2 常用代码仓

昇腾生态中有多个代码仓，各自承担不同的角色。了解它们的分工，有助于你在开发过程中快速找到需要的资源和参考。

代码仓说明开发者关系
ops-blas线性代数算子仓，实现 BLAS 标准接口（如 swap、axpy、gemm 等）BLAS 类算子的开发主阵地
ops-nn神经网络算子仓，实现常见的 NN 算子（如 Softmax、LayerNorm 等）NN 类算子的开发
ops-math数学运算算子仓，实现各类数学函数（如 Exp、Log、Sqrt 等）数学类算子的开发
ops-cv计算机视觉算子仓，实现图像处理相关算子（如 Resize、Crop 等）CV 类算子的开发
ops-transformerTransformer 专用算子仓，实现注意力机制、位置编码等 Transformer 特有算子大模型相关算子的开发
cann-samples官方算子样例仓，包含各类算子的完整实现示例学习参考，找模板和范例代码
asc-devkitAscend C SDK，包含 API 文档、示例代码和构建系统查阅 API 文档和示例代码

这些代码仓之间的关系可以这样理解：各个 ops-* 仓按算子类型分工；cann-samples 和 asc-devkit 是参考资料库，帮你快速上手。

3.3.3 框架对接

算子开发完成后，需要对接到上层框架才能在模型中使用。目前主要通过 PyTorch 对接：

(1) PyTorch（通过 torch_npu）

torch_npu 是 PyTorch 的昇腾适配插件，类似于 CUDA 版本的 PyTorch。它让 PyTorch 能够识别昇腾 NPU 设备（torch.npu），并自动将算子调度到 NPU 上执行。

对接流程：算子代码（Ascend C）→ 编译为动态库 → 注册到 ACL Runtime → torch_npu 通过 ACL 调用算子 → PyTorch 模型中使用。

3.3.4 开发工具链

CANN Toolkit 是算子开发的核心工具包，包含了编译、调试、性能分析所需的全部工具：

工具用途使用场景
CANN Toolkit开发工具包（包含以下所有工具）安装后获得完整的开发环境
ascendc 编译器将 Ascend C 代码编译为 NPU 可执行的二进制编译算子代码
msprof性能分析器，采集算子执行耗时、带宽、AI Core 利用率性能调优，定位瓶颈
cannsim 仿真器在无 NPU 硬件的环境下模拟算子执行功能验证、精度调试
ACL Runtime运行时库，管理设备、内存、流等Host 侧代码调用

开发工具的典型使用流程：

1. 安装 CANN Toolkit → 获得编译器和开发头文件
2. 编写算子代码（Ascend C）
3. 用 ascendc 编译器编译 → 生成 .o 和 .so 文件
4. 编写测试代码，链接 ACL Runtime → 在 NPU 上运行验证
5. 用 msprof 采集性能数据 → 分析瓶颈并优化
6. 迭代 2~5 直到功能和性能都达标

Q：没有 NPU 硬件也能开发算子吗？

A：可以。CANN 提供了 cannsim 仿真器，可以在 CPU 上模拟 NPU 的执行行为。cannsim 支持功能验证和精度调试，但性能数据不准确（毕竟是模拟的）。正式的性能调优还是需要在真实 NPU 上进行。
