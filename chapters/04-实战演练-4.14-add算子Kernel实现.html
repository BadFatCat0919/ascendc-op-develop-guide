4.10 add 算子 Kernel 实现

4.10.1 数据流概览

add 算子的 Kernel 实现遵循经典的搬入 → 计算 → 搬出三步走模式。下图展示了数据在各存储层级之间的流动：

4.10.2 Kernel 类结构

Kernel 类采用 Init → Process 的两阶段结构。Process 内部按顺序执行 CopyIn、Compute、CopyOut：

// add_kernel.cpp
#include "kernel_operator.h"
#include "add_tiling_data.h"

class AddKernel {
public:
    __aicore__ inline AddKernel() {}

    __aicore__ inline void Init(GM_ADDR a, GM_ADDR b, GM_ADDR c,
                                 GM_ADDR tilingGm)
    {
        // 解析 Tiling 数据
        __gm__ AddTilingData* tilingPtr =
            reinterpret_cast<__gm__ AddTilingData*>(tilingGm);
        totalLength_ = tilingPtr->totalLength;
        tileSize_ = tilingPtr->tileSize;
        alignSize_ = tilingPtr->alignSize;

        // 计算当前核的数据范围
        blockIdx_ = AscendC::GetBlockIdx();
        offset_ = blockIdx_ * tileSize_;

        // 最后一个核可能处理的数据不足 tileSize，需要截断
        processLength_ = tileSize_;
        if (offset_ + tileSize_ > totalLength_) {
            processLength_ = totalLength_ - offset_;
        }

        // 初始化 GM 地址
        aGm_ = reinterpret_cast<__gm__ float*>(a) + offset_;
        bGm_ = reinterpret_cast<__gm__ float*>(b) + offset_;
        cGm_ = reinterpret_cast<__gm__ float*>(c) + offset_;

        // 分配 UB Buffer（双 Buffer 流水线）
        pipe_.InitBuffer(inQueueA_, 2, alignSize_ * sizeof(float));
        pipe_.InitBuffer(inQueueB_, 2, alignSize_ * sizeof(float));
        pipe_.InitBuffer(outQueue_, 1, alignSize_ * sizeof(float));
    }

    __aicore__ inline void Process()
    {
        CopyIn();
        Compute();
        CopyOut();
    }

private:
    // ... 成员方法见下文

    // 成员变量
    AscendC::TPipe pipe_;
    AscendC::TQue<AscendC::QuePosition::VECIN, 1> inQueueA_;
    AscendC::TQue<AscendC::QuePosition::VECIN, 1> inQueueB_;
    AscendC::TQue<AscendC::QuePosition::VECOUT, 1> outQueue_;

    __gm__ float* aGm_ = nullptr;
    __gm__ float* bGm_ = nullptr;
    __gm__ float* cGm_ = nullptr;

    uint32_t totalLength_ = 0;
    uint32_t tileSize_ = 0;
    uint32_t alignSize_ = 0;
    uint32_t processLength_ = 0;
    uint32_t blockIdx_ = 0;
    uint32_t offset_ = 0;
};

4.10.3 数据搬运流程

CopyIn：从 GM 搬入到 UB

__aicore__ inline void CopyIn()
{
    // 从 GM 搬入数据到 UB
    auto aLocal = inQueueA_.AllocTensor<float>();
    auto bLocal = inQueueB_.AllocTensor<float>();
    AscendC::DataCopy(aLocal, aGm_, alignSize_);
    AscendC::DataCopy(bLocal, bGm_, alignSize_);
    inQueueA_.EnQue(aLocal);
    inQueueB_.EnQue(bLocal);
}

要点：

  • 使用 DataCopy 从 GM 搬入 A 和 B 两个向量到 UB
  • 搬入长度使用 alignSize（32B 对齐），即使实际数据不足也要搬对齐长度
  • 搬入后通过 EnQue 入队，供计算阶段使用

Compute：Vector 计算

__aicore__ inline void Compute()
{
    // 从队列取出数据，执行加法
    auto aLocal = inQueueA_.DeQue<float>();
    auto bLocal = inQueueB_.DeQue<float>();
    auto cLocal = outQueue_.AllocTensor<float>();

    AscendC::Add(cLocal, aLocal, bLocal, processLength_);

    outQueue_.EnQue(cLocal);
    inQueueA_.FreeTensor(aLocal);
    inQueueB_.FreeTensor(bLocal);
}

要点：

  • 通过 DeQue 从输入队列取出数据
  • 调用 AscendC::Add 执行逐元素加法
  • 计算长度使用 processLength（实际数据长度），而非 alignSize
  • 计算完成后释放输入 Tensor，结果入队

CopyOut：从 UB 搬回 GM

__aicore__ inline void CopyOut()
{
    // 将结果从 UB 搬回 GM
    auto cLocal = outQueue_.DeQue<float>();
    AscendC::DataCopy(cGm_, cLocal, processLength_);
    outQueue_.FreeTensor(cLocal);
}

搬出长度同样使用 processLength，只搬出有效数据。

4.10.4 Kernel 入口函数

Kernel 入口函数是 AI Core 的执行入口，使用 __global__ 和 __aicore__ 标记：

// Kernel 入口函数
extern "C" __global__ __aicore__ void add_kernel(
    GM_ADDR a, GM_ADDR b, GM_ADDR c,
    GM_ADDR workspace, GM_ADDR tiling)
{
    AddKernel op;
    op.Init(a, b, c, tiling);
    op.Process();
}

搬入和计算/搬出的长度不同：DataCopy 要求 32B 对齐，所以搬入用 alignSize；但实际计算和搬出只需要 processLength 个元素。这是 add 算子中最容易踩的坑之一。

Vector 的 repeat + mask 机制让它能处理任意长度的向量，不需要程序员手写循环。AscendC::Add 内部自动计算 repeat 次数和最后一次 repeat 的 mask。

Q&A

Q：Init 阶段为什么要检查 offset_ + tileSize_ > totalLength_？

A：当 totalLength 不能被 coreNum 整除时，最后一个核分到的数据量不足 tileSize。如果不做截断，这个核会访问超出向量边界的数据，导致越界。所以需要计算 processLength = totalLength - offset 来截断。

Q：为什么 UB Buffer 用双 Buffer（InitBuffer 的第二个参数为 2）？

A：双 Buffer 支持流水线：当一个 Buffer 在做计算时，另一个 Buffer 可以同时搬入下一批数据。虽然 add 算子只有一轮搬入→计算→搬出，双 Buffer 的优势不明显，但在多轮循环的场景下（大数据量分块处理），双 Buffer 能显著提升吞吐。

Q：如果 processLength = 500，Vector 宽度为 256 Bytes（64 个 float），需要多少次 repeat？

A：ceil(500 / 64) = 8 次 repeat。前 7 次每次处理 64 个元素（共 448 个），第 8 次通过 mask 机制只处理剩余的 52 个元素。
