4.1 算子精度标准

4.1.1 算子精度标准的定义

算子精度标准是衡量 NPU 算子输出结果与"标准答案"之间偏差的一套量化指标体系。它回答了一个核心问题：我的算子算出来的结果，到底够不够准？

由于不同硬件的浮点运算实现存在差异，同样的算子在 NPU 和 CPU（或其他芯片）上运行，结果无法做到完全一致。精度标准定义了"偏差在什么范围内可以接受"，是算子开发中不可绕过的验收门槛。

精度标准之所以重要，有三个原因：

  1. 保证模型正确性：算子是模型的最小执行单元，算子精度不达标会导致模型训练发散或推理结果错误。
  2. 统一验收标准：有了量化指标，开发者可以明确判断算子是否达到交付要求，而不是凭感觉"差不多就行"。
  3. 指导开发决策：精度标准直接影响 Tiling 策略、计算精度选择和 Buffer 规划。了解标准，才能在开发初期就做出正确的技术选型。

商用精度标准根据算子的计算特性，将算子分为四类，每类适用不同的精度评估方法：

算子类别典型算子评估方法
非计算类Copy、Swap、Cast、Transpose二进制一致（Bitwise Match）
整数计算类整数加减、位运算二进制一致 + 绝对误差
量化计算类Quantize、Dequantize绝对误差（整型输出）/ 双标杆 Ratio（浮点输出）
浮点计算类Add、Mul、Matmul、Softmax双标杆 Ratio（MARE / MERE / RMSE）

本章重点介绍商用精度标准的核心指标和使用方法。

4.1.2 商用精度标准详解

(1) 精度等级：L0 / L1 / L2

商用标准基于算子重要性，设计了从 L0 到 L2 的三级精度看护体系。等级越高，测试规模越大，通过阈值越严格：

精度等级等级定义用例规模MARE RatioMERE RatioRMSE Ratio适用场景
L0常规算子≥ 5,000≤ 10≤ 2≤ 2非敏感业务
L1重要算子≥ 10,000≤ 5≤ 1.5≤ 1.5LLM / 多模态 / 推荐系统
L2关键算子≥ 30,000≤ 2≤ 1.2≤ 1.2核心业务关键算子

开发算子时，首先要确认算子的精度等级。等级越高，开发难度越大——不仅要保证计算逻辑正确，还要在大规模用例下保持精度稳定。

(2) 核心误差指标

浮点计算类算子使用三个核心误差指标来衡量精度：

MARE（Max Absolute Relative Error，最大绝对相对误差）：所有输出元素中，相对误差的最大值。它反映的是"最坏情况下偏差有多大"：

MARE = max(|actual - golden| / (|golden| + 1e-7))

MERE（Mean Error of Relative Error，平均相对误差）：所有输出元素的相对误差的平均值。它反映的是"整体偏差水平"：

MERE = avg(|actual - golden| / (|golden| + 1e-7))

RMSE（Root Mean Squared Error，均方根误差）：衡量误差的分布情况（集中还是分散）：

RMSE = sqrt(mean((actual - golden)^2))

关于 1e-7：计算相对误差时，分母加上 1e-7 是为了避免 golden 值为 0 时出现除零错误。这个值足够小，不会影响正常值域的误差计算。

(3) 双标杆比对方法

商用标准采用双标杆比对，这是它与社区标准（单标杆）的核心区别。

具体做法：以更高精度的 CPU 实现作为"真值（Golden）"，同时评估三方芯片实现和 NPU 实现相对于真值的误差，然后计算两者的比值（Ratio）：

Ratio = NPU 误差指标 / 三方芯片误差指标

例如，MARE Ratio 的计算过程：

MARE_npu     = max(|npu_output - golden| / (|golden| + 1e-7))
MARE_benchmark = max(|benchmark_output - golden| / (|golden| + 1e-7))
MARE_Ratio    = MARE_npu / MARE_benchmark

Ratio 的含义是：NPU 的误差是三方芯片误差的多少倍。Ratio ≤ 阈值，意味着 NPU 的精度没有比三方芯片差太多，在可接受范围内。

标杆选择优先级：

优先级标杆类型适用场景
1业界 CPU / 三方芯片同等功能算子标准算子（如 BLAS）
2小算子拼接组合实现融合算子
3自行构造的 CPU 实现非标准数据类型

没有三方芯片对标时怎么办？对于自定义算子或非标准算子，可以使用小算子拼接实现或自行构造的高精度 CPU 实现作为标杆。

(4) 非计算类与整数计算类标准

非计算类算子（如 Copy、Swap、Cast）不涉及数值计算，输出应该是输入的精确副本或确定性变换。通过标准最简单也最严格：二进制一致（Bitwise Match）。

is_pass = np.array_equal(npu_output, golden_output)

只要有一个 bit 不同，就算失败。没有"差不多就行"的空间。

整数计算类算子（如整数加减、位运算）同样要求二进制一致。整数运算没有舍入误差，结果应该是确定性的。如果整型算子的输出和 golden 不一致，那一定是代码有 bug，而不是精度问题。

(5) 小值域与特殊值处理

当 golden 值接近 0 时，相对误差的计算会变得不稳定（分母太小，一点偏差就会被放大）。此时需要启用小值域标准：

数据类型小值阈值误差阈值
FP322-142-30
FP162-112-16
BF162-82-16
HiFP322-122-28

小值域标准使用 ErrorCount（超出误差阈值的元素个数）来评估，通过标准是 NPU 的 ErrorCount 与三方芯片的 ErrorCount 之比 ≤ 2。

当输出包含 INF 或 NAN 时，需要特殊处理：如果 golden 是 INF/NAN，NPU 输出也应该是对应的 INF/NAN。具体的 INF/NAN 处理规则取决于芯片架构和环境变量配置。

4.1.3 如何在开发中使用精度标准

(1) 开发阶段的精度检查流程

在算子开发过程中，精度验证通常遵循以下步骤：

  1. 确认精度等级：根据算子的业务重要性，确定 L0/L1/L2 等级。
  2. 准备标杆：准备 CPU golden 实现和三方芯片标杆（或替代方案）。
  3. 生成测试用例：按照精度等级要求的规模和分布生成测试数据。
  4. 执行并比对：在 NPU 上运行算子，计算 MARE/MERE/RMSE Ratio。
  5. 判定结果：Ratio 全部 ≤ 阈值则通过，否则进入精度调试。

(2) 精度验证代码示例

以下是一个浮点计算类算子的精度验证示例（Python）：

import numpy as np

def compute_mare(actual, golden):
    """计算最大绝对相对误差"""
    return np.max(np.abs(actual - golden) / (np.abs(golden) + 1e-7))

def compute_mere(actual, golden):
    """计算平均相对误差"""
    return np.mean(np.abs(actual - golden) / (np.abs(golden) + 1e-7))

def compute_rmse(actual, golden):
    """计算均方根误差"""
    return np.sqrt(np.mean((actual - golden) ** 2))

def check_precision(npu_out, golden, benchmark_out, level='L1'):
    """双标杆精度检查"""
    thresholds = {
        'L0': {'mare': 10, 'mere': 2, 'rmse': 2},
        'L1': {'mare': 5,  'mere': 1.5, 'rmse': 1.5},
        'L2': {'mare': 2,  'mere': 1.2, 'rmse': 1.2},
    }
    th = thresholds[level]

    # 计算 NPU 和标杆的误差指标
    mare_npu = compute_mare(npu_out, golden)
    mare_bmk = compute_mare(benchmark_out, golden)
    mere_npu = compute_mere(npu_out, golden)
    mere_bmk = compute_mere(benchmark_out, golden)
    rmse_npu = compute_rmse(npu_out, golden)
    rmse_bmk = compute_rmse(benchmark_out, golden)

    # 计算 Ratio
    mare_ratio = mare_npu / max(mare_bmk, 1e-10)
    mere_ratio = mere_npu / max(mere_bmk, 1e-10)
    rmse_ratio = rmse_npu / max(rmse_bmk, 1e-10)

    # 判定
    passed = (mare_ratio <= th['mare'] and
              mere_ratio <= th['mere'] and
              rmse_ratio <= th['rmse'])

    return {
        'is_pass': passed,
        'mare_ratio': mare_ratio,
        'mere_ratio': mere_ratio,
        'rmse_ratio': rmse_ratio,
    }

对于非计算类算子，验证更简单——直接做二进制比对：

def check_bitwise(npu_out, golden):
    """非计算类算子精度检查：二进制一致"""
    return {'is_pass': np.array_equal(npu_out, golden)}

(3) 精度不达标时的复检

当单个用例不满足通过标准时，不要急着判定失败。NPU 执行存在一定的随机性，单次运行的结果可能受数值不稳定性影响。

此时可以启动复检流程：

  1. 更换不同的随机种子，重新生成输入并执行 N 次（推荐 1000 次）。
  2. 计算每次运行的 Ratio。
  3. 使用 Bootstrap 方法计算 Ratio 中位数的 95% 置信区间。
  4. 如果置信区间下限 > 1.0，才判定为真正失败。

复检的核心思想是：用统计学方法排除偶发的数值波动，确认精度问题是系统性的还是随机性的。

注意：如果重新执行的次数 N < 200，样本量不足以支撑统计推断，直接判定为不通过。
