5.3 计算 API 详解

本节详细介绍 Ascend C 的计算类 API，包括向量算术运算、向量数学函数、标量运算、归约运算、类型转换和矩阵计算。这些 API 是算子 Kernel 中执行实际计算的核心接口。

5.3.1 向量算术运算

向量算术运算 API 对两个输入 Tensor 的对应元素执行二元运算，由 Vector 单元执行。所有 API 共享相同的调用模式。

5.3.1.1 通用签名

template <typename T>
__aicore__ inline void Add(const LocalTensor<T>& dst,
                            const LocalTensor<T>& src0,
                            const LocalTensor<T>& src1,
                            const uint32_t calCount,
                            const uint8_t repeatTimes,
                            const RepeatParams& repeatParams);
参数类型说明
dstLocalTensor<T>输出 Tensor（UB 上）
src0LocalTensor<T>第一个输入 Tensor
src1LocalTensor<T>第二个输入 Tensor
calCountuint32_t要处理的元素总数
repeatTimesuint8_t向量指令重复次数（最大 255）
repeatParamsRepeatParams源和目标的地址步进参数

5.3.1.2 RepeatParams 结构

struct RepeatParams {
    uint16_t dstRepStride;
    uint8_t src0RepStride;
    uint8_t src1RepStride;
    uint16_t dstBlkStride;
    uint8_t src0BlkStride;
    uint8_t src1BlkStride;
};
字段说明
dstRepStride目标 Tensor 的 repeat 间步长（单位：128 字节）
src0RepStride源 Tensor 0 的 repeat 间步长
src1RepStride源 Tensor 1 的 repeat 间步长
dstBlkStride目标 Tensor 的块内步长
src0BlkStride源 Tensor 0 的块内步长
src1BlkStride源 Tensor 1 的块内步长

5.3.1.3 repeat 次数计算

每次 repeat 处理 256 字节数据。repeat 次数的计算公式：

repeatTimes = ceil(calCount * sizeof(T) / 256)
数据类型每 repeat 元素数示例：1024 元素
FP16 / BF161281024 / 128 = 8 次 repeat
FP32 / INT32641024 / 64 = 16 次 repeat
INT82561024 / 256 = 4 次 repeat

5.3.1.4 常用算术 API 对照

API运算数学表达
Add逐元素加dst[i] = src0[i] + src1[i]
Sub逐元素减dst[i] = src0[i] - src1[i]
Mul逐元素乘dst[i] = src0[i] * src1[i]
Div逐元素除dst[i] = src0[i] / src1[i]
Max逐元素取大dst[i] = max(src0[i], src1[i])
Min逐元素取小dst[i] = min(src0[i], src1[i])

5.3.1.5 使用示例

RepeatParams rp = {1, 1, 1, 8, 8, 8};
Add(dst, src0, src1, 1024, 8, rp);
Sub(dst, src0, src1, 1024, 8, rp);
Mul(dst, src0, src1, 512, 4, rp);

5.3.2 向量数学函数

向量数学函数对单个输入 Tensor 的每个元素执行一元运算。

5.3.2.1 通用签名（一元运算）

template <typename T>
__aicore__ inline void Exp(const LocalTensor<T>& dst,
                            const LocalTensor<T>& src,
                            const uint32_t calCount,
                            const uint8_t repeatTimes,
                            const RepeatParams& repeatParams);
API运算数学表达输入类型要求
Abs绝对值dst[i] = |src[i]|FP16/FP32/BF16
Exp指数dst[i] = e^src[i]FP16/FP32
Ln自然对数dst[i] = ln(src[i])FP16/FP32
Sqrt平方根dst[i] = sqrt(src[i])FP16/FP32
Reciprocal倒数dst[i] = 1/src[i]FP16/FP32

5.3.2.2 使用示例

RepeatParams rp = {1, 1, 8, 8};
Exp(dst, src, 128, 1, rp);
Sqrt(dst, src, 256, 2, rp);
Abs(dst, src, 128, 1, rp);

5.3.3 标量运算

标量运算 API 将一个向量与一个标量值进行运算，常用于缩放、偏移等操作。

5.3.3.1 Adds

template <typename T>
__aicore__ inline void Adds(const LocalTensor<T>& dst,
                             const LocalTensor<T>& src,
                             const T scalarValue,
                             const uint32_t calCount,
                             const uint8_t repeatTimes,
                             const RepeatParams& repeatParams);

5.3.3.2 Muls

template <typename T>
__aicore__ inline void Muls(const LocalTensor<T>& dst,
                             const LocalTensor<T>& src,
                             const T scalarValue,
                             const uint32_t calCount,
                             const uint8_t repeatTimes,
                             const RepeatParams& repeatParams);
API运算数学表达
Adds向量 + 标量dst[i] = src[i] + scalar
Muls向量 × 标量dst[i] = src[i] * scalar

5.3.3.3 使用示例

RepeatParams rp = {1, 1, 8, 8};
Adds(dst, src, 1.0f, 128, 1, rp);
Muls(dst, src, 0.5f, 256, 2, rp);

5.3.4 归约运算

归约运算将多个元素聚合为更少的结果。Ascend C 提供三种基本归约操作。

5.3.4.1 函数签名

template <typename T>
__aicore__ inline void ReduceSum(const LocalTensor<T>& dst,
                                  const LocalTensor<T>& src,
                                  const LocalTensor<T>& work,
                                  const uint32_t calCount);
API功能输出
ReduceSum求和dst[0] = sum(src[0..N-1])
ReduceMax求最大值dst[0] = max(src[0..N-1])
ReduceMin求最小值dst[0] = min(src[0..N-1])

归约 API 需要一个 work Buffer 用于中间计算结果。work Buffer 的大小取决于数据类型和 calCount。

5.3.4.2 两阶段归约模式

在多核场景中，归约通常采用两阶段模式：

  1. 核内归约：每个 AI Core 对自己的 tile 做局部归约
  2. 跨核归约：所有核将中间结果写入 GM workspace，由一个核做全局归约

5.3.5 类型转换（Cast）

5.3.5.1 函数签名

template <typename DST_T, typename SRC_T>
__aicore__ inline void Cast(const LocalTensor<DST_T>& dst,
                             const LocalTensor<SRC_T>& src,
                             const uint32_t calCount,
                             const uint8_t repeatTimes,
                             const RepeatParams& repeatParams);

5.3.5.2 支持的转换路径

源类型目标类型精度影响
FP32FP16精度损失（10 位尾数）
FP32BF16精度损失（7 位尾数）
FP16FP32无损
BF16FP32无损
FP16BF16需要先转 FP32 中转
FP32INT8/INT32截断
INT8/INT32FP32无损

5.3.5.3 精度注意事项

FP16 → BF16 不能直接转换，因为两者的指数位宽度不同。正确做法是先 Cast 到 FP32（保留完整精度），再从 FP32 Cast 到 BF16：

Cast(tmpFp32, srcFp16, count, repeat, rp);
Cast(dstBf16, tmpFp32, count, repeat, rp);

5.3.6 矩阵计算

5.3.6.1 Matmul

template <typename T>
__aicore__ inline void Matmul(const LocalTensor<T>& dstL0C,
                               const LocalTensor<T>& srcL0A,
                               const LocalTensor<T>& srcL0B,
                               const MatmulConfig& config);

Matmul 驱动 Cube 单元执行矩阵乘法 C = A × B。输入数据必须在 L0A 和 L0B 中，输出在 L0C 中。矩阵维度必须是 16 的倍数（FP16）。

5.3.6.2 MatmulLeakyRelu

template <typename T>
__aicore__ inline void MatmulLeakyRelu(const LocalTensor<T>& dstL0C,
                                        const LocalTensor<T>& srcL0A,
                                        const LocalTensor<T>& srcL0B,
                                        const MatmulConfig& config,
                                        const float alpha);

将矩阵乘法与 LeakyReLU 激活函数融合为一条指令，减少后处理开销。alpha 是 LeakyReLU 的负半轴斜率。

5.3.6.3 FixPipe

template <typename DST_T, typename SRC_T>
__aicore__ inline void FixPipe(const LocalTensor<DST_T>& dst,
                                const LocalTensor<SRC_T>& src,
                                const uint32_t calCount,
                                const uint8_t repeatTimes,
                                const FixPipeParams& params);

FixPipe 对 Cube 输出（L0C）做后处理，包括精度转换（FP32→FP16）、量化（浮点→INT8）等。它是 Cube 流水线的最后一个阶段。

Q&A

Q：repeatTimes 的最大值是多少？超过怎么办？

A：repeatTimes 是 uint8_t 类型，最大值为 255。如果需要处理的数据超过 255 次 repeat 的容量（255 × 256B = 65280B），需要拆分为多次调用。例如处理 2048 个 FP32 元素（8192B = 32 次 repeat），不需要拆分；但处理 20000 个 FP32 元素（80000B ≈ 313 次 repeat），需要拆成两次调用。

Q：RepeatParams 中的 stride 参数如何设置？

A：在最简单的连续数据场景中，repStride 设为 repeatTimes 的值（或 1，取决于数据布局），blkStride 设为 8。具体取值取决于 UB 中数据的排列方式。如果数据是连续排列的，通常 repStride = repeatTimes，blkStride = 8（表示块内步长为 8 个 128B 块 = 1024B = 4 次 repeat 的容量）。

Q：为什么 FP16 → BF16 不能直接转换？

A：FP16 和 BF16 的指数位宽度不同（FP16 有 5 位指数，BF16 有 8 位指数），尾数位也不同（FP16 有 10 位，BF16 有 7 位）。直接转换会丢失指数范围信息。正确做法是先转 FP32（保留完整精度），再从 FP32 转 BF16。
