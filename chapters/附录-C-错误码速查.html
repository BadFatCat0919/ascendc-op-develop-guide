Ascend C 算子从 Host 侧调用到 Kernel 执行，涉及多层软件栈。每一层都有自己的错误码体系。本表汇总了开发中最常遇到的错误码及其含义。

ACL 基础错误码

这是最底层的运行时错误码，由 acl 系列 API 返回：

错误码枚举名含义常见原因
0ACL_SUCCESS成功—
100001ACL_ERROR_INTERNAL内部错误驱动层异常，查看 plog 定位
100002ACL_ERROR_MALLOC_FAILURE内存分配失败设备显存不足
100003ACL_ERROR_INVALID_PARAM参数无效传入了空指针或非法值
100004ACL_ERROR_INVALID_DEVICE设备无效deviceId 超出范围
100006ACL_ERROR_INVALID_RESOURCE资源无效Stream/Event 已释放
100015ACL_ERROR_FEATURE_UNSUPPORTED特性不支持当前芯片不支持该功能

aclblas 错误码

aclblas 是 ops-blas 仓定义的上层错误码，在 ACL 基础错误码之上增加了 BLAS 语义：

错误码枚举名含义常见原因
0ACLBLAS_STATUS_SUCCESS成功—
1ACLBLAS_STATUS_NOT_INITIALIZED未初始化未调用 aclblasCreate
2ACLBLAS_STATUS_ALLOC_FAILED内存分配失败aclrtMalloc 返回失败
3ACLBLAS_STATUS_INVALID_VALUE参数值无效n<0、incx=0、指针为空等
4ACLBLAS_STATUS_MAPPING_ERROR映射错误算子 Kernel 查找失败
5ACLBLAS_STATUS_EXECUTION_FAILED执行失败Kernel 运行时异常
6ACLBLAS_STATUS_INTERNAL_ERROR内部错误aclrtMemcpy 等底层 API 失败
7ACLBLAS_STATUS_NOT_SUPPORTED不支持数据类型或架构不支持
8ACLBLAS_STATUS_HANDLE_IS_NULLPTRHandle 为空传入了 nullptr 的 handle

sswap 参数校验示例

sswap 的 Host 侧代码展示了参数校验和错误码返回的典型模式：

aclblasStatus_t aclblasSswap(aclblasHandle_t handle, int n,
                              float* x, int incx, float* y, int incy)
{
    // n <= 0 时直接返回成功（BLAS 标准行为）
    if (n <= 0) {
        return ACLBLAS_STATUS_SUCCESS;
    }

    // handle 为空 → ACLBLAS_STATUS_HANDLE_IS_NULLPTR
    if (handle == nullptr) {
        return ACLBLAS_STATUS_HANDLE_IS_NULLPTR;
    }

    // x/y 为空、incx/incy 为 0 或不等于 1 → ACLBLAS_STATUS_INVALID_VALUE
    aclblasStatus_t st = ValidateSswapParams(x, y, incx, incy);
    if (st != ACLBLAS_STATUS_SUCCESS) {
        return st;
    }

    // 获取核数失败 → ACLBLAS_STATUS_EXECUTION_FAILED
    uint32_t aivCoreNum = GetVectorCoreCount();
    if (aivCoreNum == 0) {
        return ACLBLAS_STATUS_EXECUTION_FAILED;
    }

    // 内存分配失败 → ACLBLAS_STATUS_ALLOC_FAILED
    // 内存拷贝失败 → ACLBLAS_STATUS_INTERNAL_ERROR
    ...
}

Tiling 相关错误（161xxx 范围）

Tiling 阶段的错误通常以 161 开头：

错误码范围含义常见原因
161001~161099Tiling 参数校验失败TilingData 中的值超出合法范围
161100~161199Tiling 数据传递失败aclrtMemcpy H2D 拷贝 TilingData 失败
161200~161299Tiling 函数执行异常Tiling 函数内部逻辑错误

Kernel 查找与加载错误（361xxx 范围）

Kernel 查找阶段的错误通常以 361 开头：

错误码范围含义常见原因
361001~361099Kernel 查找失败算子未注册、架构不匹配
361100~361199Kernel 加载失败Kernel 二进制损坏、驱动版本不匹配
361200~361299Kernel 参数校验失败GM 地址无效、TilingData 格式不对

Kernel 运行时错误（561xxx 范围）

Kernel 在 AI Core 上执行时的错误：

错误码范围含义常见原因
561001~561099数据搬运异常DataCopy 长度不对齐、地址越界
561100~561199计算指令异常Vector/Cube 指令参数非法
561200~561299同步异常SetFlag/WaitFlag 死锁、Queue 溢出
561300~561399内存越界UB 访问超出分配范围

排查流程

遇到错误时，推荐按以下流程排查：

  1. 看错误码范围，确定是哪一层出了问题（Host / Tiling / Kernel 查找 / Kernel 运行）。
  2. 查看 plog 日志，搜索 error 或 fail 关键词，定位具体的错误位置。
  3. 检查参数，确认传入的指针、维度、数据类型是否合法。
  4. 检查对齐，确认 DataCopy 长度是否满足 32B 对齐要求。
  5. 检查资源，确认设备显存是否充足、Stream 是否正常。
