4.2a 获取与编译算子代码

swap 是 BLAS（Basic Linear Algebra Subprograms）标准中最简单的算子之一：交换两个向量的对应元素。我们用它作为第一个实战项目，因为它逻辑简单但结构完整，能让你快速理解算子开发的全流程。本节聚焦于获取代码和编译两个步骤。

克隆 ops-blas 仓库

# 克隆 ops-blas 算子仓库
git clone https://gitcode.com/cann/ops-blas.git
cd ops-blas

# 查看仓库结构
ls -la
# 输出：
# blas/          # 算子源码目录
# build.sh       # 编译脚本
# test/          # 测试用例
# CMakeLists.txt # CMake 配置
# ...

定位 swap 算子代码

# swap 算子位于 blas/swap/ 目录
ls blas/swap/
# 输出：
# arch35/        # 针对 A5 架构的实现
# CMakeLists.txt

# 查看 arch35 目录下的文件
ls blas/swap/arch35/
# 输出：
# swap_host.cpp       # Host 侧实现
# swap_kernel.cpp     # Kernel 侧实现
# swap_tiling_data.h  # Tiling 数据结构定义

代码文件说明

文件作用运行位置
swap_host.cpp参数校验、Tiling 计算、启动 KernelCPU（Host 侧）
swap_kernel.cppAscend C 核函数实现NPU（Device 侧）
swap_tiling_data.hHost 和 Kernel 共享的 Tiling 数据结构编译时共用

Q&A：为什么按架构分目录？

Q：为什么 swap 算子的代码要放在 arch35/ 子目录下，而不是直接放在 blas/swap/ 下？

A：不同 NPU 架构（如 A5 对应 arch35）在指令集、寄存器数量、UB 大小等方面存在差异。按架构分目录可以让同一算子针对不同硬件做针对性优化，同时保持代码组织清晰。编译时 --soc 参数会自动选择对应架构目录。

编译命令详解

ops-blas 仓库提供了统一的编译脚本 build.sh，一行命令就能编译指定算子。

编译命令

# 编译 swap 算子，目标芯片为 Ascend 910B（架构代号 ascend950）
bash build.sh --ops=swap --soc=ascend950

# 编译过程输出（简化）：
# [INFO] Building swap for ascend950...
# [INFO] Compiling swap_kernel.cpp...
# [INFO] Compiling swap_host.cpp...
# [INFO] Linking...
# [INFO] Build success!
# [INFO] Output: out/swap/libswap.so

编译参数说明

参数说明示例值
--ops要编译的算子名称swap、gemm、axpy
--soc目标芯片型号ascend950（910B）、ascend910
--debug开启调试模式（含调试符号）添加此标志
--clean清理后重新编译（全量编译）添加此标志

编译产物

# 编译成功后，产物在 out/ 目录
ls out/swap/
# 输出：
# libswap.so           # 动态库（算子实现）
# swap_kernel.o        # Kernel 目标文件
# swap_host.o          # Host 目标文件

# 同时会生成编译日志
cat out/swap/build.log

Q&A：编译常见问题

Q：编译时报 undefined reference to 'DataCopy' 怎么办？

A：这通常是头文件包含不完整导致的。确保 Kernel 文件中 #include "kernel_operator.h" 已正确引入。如果 CANN 环境未正确 source，也可能导致编译器找不到 Ascend C API。

Q：--soc 参数填错了会怎样？

A：编译可能仍然成功，但生成的二进制只能在对应架构的芯片上运行。如果部署到不匹配的芯片上，会出现 Kernel 查找失败（错误码 561xxx）。务必确认目标芯片型号。
