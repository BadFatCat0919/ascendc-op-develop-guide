4.1a CANN 安装与配置

本节聚焦 CANN Toolkit 的下载、安装、环境变量配置和安装验证。完成本节后，你将拥有一个可用的 CANN 开发环境。

图：CANN Toolkit 安装四步流程

CANN Toolkit 下载

CANN（Compute Architecture for Neural Networks）是昇腾 NPU 的统一计算架构，Toolkit 里包含了编译器、运行时库和各种开发工具。

访问昇腾社区官网下载页面：

# 访问下载页面
https://www.hiascend.com/software/cann/community

# 选择对应版本（推荐最新稳定版）
# 下载文件名格式：Ascend-cann-toolkit_8.x.x_linux-aarch64.run（ARM）
# 或：Ascend-cann-toolkit_8.x.x_linux-x86_64.run（x86）

根据你的服务器架构选择对应的安装包：

架构适用场景安装包后缀
aarch64ARM 服务器（如 Atlas 800）linux-aarch64.run
x86_64x86 服务器 + NPU 加速卡linux-x86_64.run

安装步骤

# 1. 添加执行权限
chmod +x Ascend-cann-toolkit_8.x.x_linux-aarch64.run

# 2. 执行安装（默认安装到 ~/Ascend/ascend-toolkit）
./Ascend-cann-toolkit_8.x.x_linux-aarch64.run --install

# 3. 如果需要指定安装路径
./Ascend-cann-toolkit_8.x.x_linux-aarch64.run --install --install-path=/opt/Ascend

# 4. 安装完成后，目录结构如下
~/Ascend/ascend-toolkit/latest/
├── bin/              # 可执行工具
├── include/          # 头文件
├── lib64/            # 动态库
├── toolkit/tools/    # 开发工具
└── set_env.sh        # 环境变量脚本（重要！）

安装依赖

CANN Toolkit 依赖一些系统库，确保已安装：

# Ubuntu/Debian
sudo apt-get install -y gcc g++ make cmake python3 python3-pip

# CentOS/RHEL
sudo yum install -y gcc gcc-c++ make cmake python3 python3-pip

# 安装 Python 依赖（用于算子开发工具链）
pip3 install numpy decorator sympy cffi pyyaml pathlib psutil

环境变量配置

安装完成后，必须配置环境变量，否则编译器找不到头文件和库。

临时生效（当前终端）

# 执行环境变量脚本
source ~/Ascend/ascend-toolkit/latest/set_env.sh

# 验证环境变量是否生效
echo $ASCEND_HOME
# 应输出：/home/你的用户名/Ascend/ascend-toolkit/latest

# 检查 PATH 是否包含 CANN 工具
which ascendc
# 应输出：/home/你的用户名/Ascend/ascend-toolkit/latest/toolkit/tools/ascendc

永久生效（推荐）

# 将 source 命令添加到 ~/.bashrc
echo 'source ~/Ascend/ascend-toolkit/latest/set_env.sh' >> ~/.bashrc

# 重新加载配置
source ~/.bashrc

# 或者添加到 ~/.profile（适用于登录 shell）
echo 'source ~/Ascend/ascend-toolkit/latest/set_env.sh' >> ~/.profile

关键环境变量说明

变量名作用典型值
ASCEND_HOMECANN 安装根目录~/Ascend/ascend-toolkit/latest
PATH可执行文件搜索路径包含 $ASCEND_HOME/bin
LD_LIBRARY_PATH动态库搜索路径包含 $ASCEND_HOME/lib64
ASCEND_OPP_PATH算子包路径$ASCEND_HOME/opp

验证安装

安装完成后，运行以下命令验证：

# 1. 检查 ascendc 编译器版本
ascendc --version
# 输出示例：Ascend C Compiler version 8.x.x

# 2. 检查编译器路径
which ascendc
# 应输出 CANN 安装目录下的路径

# 3. 检查运行时库
ls $ASCEND_HOME/lib64/ | grep libascendcl
# 应看到 libascendcl.so 等文件

# 4. 检查 NPU 设备（如果有硬件）
npu-smi info
# 应显示 NPU 设备信息表格

Q&A：常见疑问

Q：set_env.sh 到底设置了哪些变量？

A：这个脚本主要设置 4 类变量：ASCEND_HOME（安装根目录）、PATH（追加 bin 和 tools 目录）、LD_LIBRARY_PATH（追加 lib64 目录）、ASCEND_OPP_PATH（算子包路径）。你可以用 cat set_env.sh 查看完整内容。

Q：可以安装到自定义路径吗？

A：可以。使用 --install-path=/your/path 参数指定。但后续配置环境变量时，需要把 set_env.sh 的路径改为对应自定义路径下的文件。

Q：aarch64 和 x86_64 安装包有什么区别？

A：两者功能完全一致，区别仅在于编译好的二进制文件针对的 CPU 架构不同。Atlas 800 等 ARM 服务器用 aarch64 版本，普通 x86 服务器配 NPU 加速卡用 x86_64 版本。用 uname -m 命令可以确认你的服务器架构。
