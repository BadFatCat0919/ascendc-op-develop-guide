4.1c 开发环境检查清单

在开始算子开发前，使用本清单逐项确认开发环境已就绪。本页提供完整检查清单、一键验证脚本和最佳实践建议。

图：开发环境检查清单涵盖 6 大项，一键脚本统一验证

完整检查清单

在开始算子开发前，确认以下项目都已就绪：

#检查项验证命令预期结果
1CANN Toolkit 已安装ascendc --version显示版本号（如 8.x.x）
2环境变量已配置echo $ASCEND_HOME非空路径
3编译器可用which ascendc有效的绝对路径
4NPU 设备可用npu-smi info显示设备信息表格
5Python 依赖已安装python3 -c "import numpy"无报错
6算子仓库已克隆ls ops-blas/显示目录结构

开发环境验证脚本

将以下脚本保存为 check_env.sh，一键检查所有环境项：

#!/bin/bash
# check_env.sh - Ascend C 开发环境一键检查脚本
# 用法: bash check_env.sh

PASS=0
FAIL=0

check() {
    local name="$1"
    local cmd="$2"
    local expect="$3"

    printf "%-30s " "$name"
    result=$(eval "$cmd" 2>&1)
    if [ $? -eq 0 ] && [ -n "$result" ]; then
        echo "✅ PASS  ($result)"
        PASS=$((PASS + 1))
    else
        echo "❌ FAIL  ($result)"
        FAIL=$((FAIL + 1))
    fi
}

echo "======================================"
echo "  Ascend C 开发环境检查"
echo "======================================"

check "CANN Toolkit 版本"   "ascendc --version 2>/dev/null | head -1"
check "ASCEND_HOME 变量"    "echo \$ASCEND_HOME"
check "ascendc 路径"        "which ascendc 2>/dev/null"
check "NPU 设备"            "npu-smi info 2>/dev/null | head -1"
check "Python3 numpy"       "python3 -c 'import numpy; print(numpy.__version__)' 2>/dev/null"
check "算子仓库"            "ls -d ops-blas 2>/dev/null"

echo "======================================"
echo "  结果: $PASS 通过, $FAIL 失败"
echo "======================================"

if [ $FAIL -gt 0 ]; then
    echo "⚠️  有 $FAIL 项未通过，请参考 4.1a/4.1b 章节排查"
    exit 1
fi

最佳实践

1. 使用虚拟环境隔离 Python 依赖

避免系统 Python 包冲突，为 CANN 开发创建专用虚拟环境：

# 创建虚拟环境
python3.9 -m venv ~/cann_env

# 激活（每次开发前先执行）
source ~/cann_env/bin/activate

# 安装依赖
pip install numpy decorator sympy cffi pyyaml pathlib psutil

2. 将环境配置写入 .bashrc 而非 .profile

.bashrc 在每个交互式 shell 启动时执行，而 .profile 仅在登录 shell 时执行。对于日常开发（打开终端），.bashrc 更可靠。

3. 多版本 CANN 共存

如果需要同时安装多个 CANN 版本，使用 --install-path 指定不同目录，通过切换 source 的 set_env.sh 路径来切换版本：

# 安装两个版本
./Ascend-cann-toolkit_8.0.0_linux-aarch64.run --install --install-path=/opt/cann-8.0
./Ascend-cann-toolkit_8.1.0_linux-aarch64.run --install --install-path=/opt/cann-8.1

# 切换版本
source /opt/cann-8.0/set_env.sh   # 使用 8.0
source /opt/cann-8.1/set_env.sh   # 使用 8.1

4. 定期检查 NPU 健康状态

开发过程中，定期用 npu-smi info 检查设备状态，特别关注：

  • 温度：超过 85°C 需要检查散热
  • 显存占用：异常高可能有残留进程
  • 设备状态：应为 Online / Ready

Q&A：常见疑问

Q：没有 NPU 硬件也能开发吗？

A：可以。使用 CANN Simulator 仿真器可以完成功能验证和精度验证。但性能测试必须在真实硬件上进行。详见 4.1b 编译器与仿真器。

Q：检查脚本某项失败后如何快速定位问题？

A：根据失败项对照排查：Toolkit 版本失败 → 检查安装是否完整；环境变量失败 → 重新 source set_env.sh；NPU 设备失败 → 检查驱动加载状态。详细排查步骤见 4.1b 常见问题排查。

Q：团队多人共用一台服务器，环境怎么管理？

A：建议每人使用独立的虚拟环境和用户目录安装 CANN。共享 NPU 设备时注意显存分配，避免同时运行大规模测试导致 OOM。
