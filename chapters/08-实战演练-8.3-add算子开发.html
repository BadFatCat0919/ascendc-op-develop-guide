8.3 add 算子开发（aclnn 注册方式）

本章以逐元素加法 add 算子为例，讲解通过 aclnn 框架注册方式开发算子的完整流程。与 8.2 swap 算子开发 中的直调方式不同，aclnn 注册方式需要实现两段式接口（GetWorkspaceSize + Execute），并通过 CMakeLists 配置将算子注册到 CANN 框架中。

8.3.1 功能说明

add 算子执行逐元素加法：

C[i] = A[i] + B[i],  i = 0, 1, ..., n-1

支持的数据类型包括 float16、float32、bfloat16、int32 等。本章以 float32 为例。

8.3.2 aclnn 两段式接口

aclnn 框架要求算子实现两个接口：

接口职责运行位置
aclnnXxxGetWorkspaceSize根据输入 shape 计算 Workspace 大小和 Tiling 参数Host（CPU）
aclnnXxxExecute使用 Tiling 参数启动 Kernel 执行Host → Device

调用方（如 PyTorch 或 MindSpore）先调用 GetWorkspaceSize 获取所需的 Workspace 大小，分配好内存后再调用 Execute 执行算子。

8.3.3 文件结构

ops-math 仓中 add 算子的目录结构如下：

op_plugin/ops/base_ops/aclops/AddCustom/
├── op_host/
│   ├── add_custom_aclnn.cpp        # aclnn 接口注册（两段式入口）
│   └── add_custom_tiling.cpp       # Tiling 函数实现
├── op_kernel/
│   └── add_custom.cpp              # Kernel 实现（AscendC 类）
├── op_tiling/
│   └── add_custom_tiling.h         # TilingData 结构体定义
└── CMakeLists.txt                  # 编译配置与算子注册
目录职责
op_host/Host 侧代码：aclnn 接口注册、参数校验、Tiling 计算
op_kernel/Device 侧代码：AscendC Kernel 类，实现计算逻辑
op_tiling/Host/Kernel 共享的 TilingData 结构体定义
CMakeLists.txt编译配置，将算子注册到 CANN 框架

8.3.4 TilingData 设计

add 算子的 TilingData 需要描述数据切分方式和 Vector API 的 repeat/mask 参数：

#ifndef ADD_CUSTOM_TILING_H
#define ADD_CUSTOM_TILING_H

#include <cstdint>

struct AddCustomTilingData {
    uint32_t totalLength;
    uint32_t alignNum;
    uint32_t tileMod;
    uint32_t tileNumPerTime;
    uint32_t tileCount;
    uint32_t tileTail;
    uint32_t coreCount;
    uint32_t coreRemainder;
};

#endif
字段含义
totalLength总元素个数
alignNum对齐元素数（32 / sizeof(dtype)）
tileMod每次迭代处理的元素数（对齐后的 tile 大小）
tileNumPerTime每个 tile 内的 repeat 次数
tileCount完整 tile 的循环次数
tileTail尾部不足一个 tile 的元素数
coreCount每个核分配的元素数
coreRemainder最后一个核的余量

8.3.5 Kernel 实现

与 swap 不同，add 算子有真正的 Vector 计算阶段。数据流为：MTE2 搬入 → Vector Add → MTE3 搬出。

8.3.5.1 类定义

class AddCustomKernel {
public:
    __aicore__ inline AddCustomKernel() {}
    __aicore__ inline void Init(GM_ADDR a, GM_ADDR b, GM_ADDR c,
                                GM_ADDR tilingGm);
    __aicore__ inline void Process();

private:
    __aicore__ inline void ParseTilingData(GM_ADDR tilingGm);
    __aicore__ inline void CopyIn(uint32_t offset, uint32_t len);
    __aicore__ inline void Compute(uint32_t len);
    __aicore__ inline void CopyOut(uint32_t offset, uint32_t len);

    TPipe pipe_;
    GlobalTensor<float> aGM_;
    GlobalTensor<float> bGM_;
    GlobalTensor<float> cGM_;
    TQue<TPosition::VECIN, 1> queueA_;
    TQue<TPosition::VECIN, 1> queueB_;
    TQue<TPosition::VECOUT, 1> queueC_;
    AddCustomTilingData tiling_;
    uint32_t myOffset_;
    uint32_t myCount_;
};

注意与 swap 的区别：

  • 使用 TQue<VECIN, 1> 和 TQue<VECOUT, 1> 分开管理输入和输出队列，因为 add 有独立的 Vector 计算阶段
  • TPipe 作为类成员（而非外部传入），因为 add 的 Kernel 结构更简单

8.3.5.2 Init：初始化与 Buffer 分配

__aicore__ inline void AddCustomKernel::Init(
    GM_ADDR a, GM_ADDR b, GM_ADDR c, GM_ADDR tilingGm)
{
    ParseTilingData(tilingGm);

    uint32_t blockIdx = GetBlockIdx();
    myOffset_ = blockIdx * tiling_.coreCount;
    myCount_ = tiling_.coreCount;
    if (blockIdx == GetBlockNum() - 1) {
        myCount_ += tiling_.coreRemainder;
    }

    aGM_.SetGlobalBuffer((__gm__ float*)a, tiling_.totalLength);
    bGM_.SetGlobalBuffer((__gm__ float*)b, tiling_.totalLength);
    cGM_.SetGlobalBuffer((__gm__ float*)c, tiling_.totalLength);

    pipe_.InitBuffer(queueA_, 2, tiling_.tileMod * sizeof(float));
    pipe_.InitBuffer(queueB_, 2, tiling_.tileMod * sizeof(float));
    pipe_.InitBuffer(queueC_, 2, tiling_.tileMod * sizeof(float));
}

add 使用双 Buffer（ping-pong），让 MTE2 搬运和 Vector 计算可以重叠执行。每个队列分配 2 个 Buffer。

8.3.5.3 CopyIn：数据搬入

__aicore__ inline void AddCustomKernel::CopyIn(uint32_t offset, uint32_t len)
{
    LocalTensor<float> aLocal = queueA_.AllocTensor<float>();
    LocalTensor<float> bLocal = queueB_.AllocTensor<float>();

    DataCopy(aLocal, aGM_[offset], len);
    DataCopy(bLocal, bGM_[offset], len);

    queueA_.EnQue(aLocal);
    queueB_.EnQue(bLocal);
}

8.3.5.4 Compute：Vector 加法

这是 add 算子的核心计算逻辑，使用 Ascend C 的 Add API：

__aicore__ inline void AddCustomKernel::Compute(uint32_t len)
{
    LocalTensor<float> aLocal = queueA_.DeQue<float>();
    LocalTensor<float> bLocal = queueB_.DeQue<float>();
    LocalTensor<float> cLocal = queueC_.AllocTensor<float>();

    uint32_t repeatTimes = len / MASK_FULL;
    uint32_t tailCount = len % MASK_FULL;

    if (repeatTimes > 0) {
        Add(cLocal, aLocal, bLocal, MASK_FULL, repeatTimes,
            {1, 1, 1, 8, 8, 8});
    }

    if (tailCount > 0) {
        uint32_t offset = repeatTimes * MASK_FULL;
        Add(cLocal[offset], aLocal[offset], bLocal[offset],
            tailCount, 1, {1, 1, 1, 8, 8, 8});
    }

    queueC_.EnQue(cLocal);
    queueA_.FreeTensor(aLocal);
    queueB_.FreeTensor(bLocal);
}

Add API 的参数说明（参见 5.3 计算 API 详解）：

参数含义示例值
dst目标 LocalTensorcLocal
src0第一个源操作数aLocal
src1第二个源操作数bLocal
mask每次迭代处理的元素数MASK_FULL (128 for float)
repeatTimes重复执行次数len / MASK_FULL
repeatParams地址步进参数 {dstRepStride, src0RepStride, src1RepStride, ...}{1, 1, 1, 8, 8, 8}

8.3.5.5 CopyOut：数据搬出

__aicore__ inline void AddCustomKernel::CopyOut(uint32_t offset, uint32_t len)
{
    LocalTensor<float> cLocal = queueC_.DeQue<float>();
    DataCopy(cGM_[offset], cLocal, len);
    queueC_.FreeTensor(cLocal);
}

8.3.5.6 Process：主循环

__aicore__ inline void AddCustomKernel::Process()
{
    if (myCount_ == 0) {
        return;
    }

    uint32_t offset = myOffset_;
    uint32_t tileLoop = myCount_ / tiling_.tileMod;
    uint32_t tileTail = myCount_ % tiling_.tileMod;

    for (uint32_t i = 0; i < tileLoop; i++) {
        CopyIn(offset, tiling_.tileMod);
        Compute(tiling_.tileMod);
        CopyOut(offset, tiling_.tileMod);
        offset += tiling_.tileMod;
    }

    if (tileTail > 0) {
        CopyIn(offset, tileTail);
        Compute(tileTail);
        CopyOut(offset, tileTail);
    }
}

8.3.5.7 Kernel 入口

extern "C" __global__ __aicore__ void add_custom_kernel(
    GM_ADDR a, GM_ADDR b, GM_ADDR c, GM_ADDR tilingGm)
{
    AddCustomKernel op;
    op.Init(a, b, c, tilingGm);
    op.Process();
}

8.3.6 Host 侧实现

8.3.6.1 Tiling 函数

Tiling 函数在 GetWorkspaceSize 阶段被调用，负责计算切分参数：

static void AddCustomTilingFunc(
    const ge::Shape& aShape, const ge::Shape& bShape,
    const ge::Shape& cShape, const op::AddCustom& op,
    AddCustomTilingData& tiling, uint32_t& workspaceSize)
{
    uint32_t totalLength = aShape.GetShapeSize();
    tiling.totalLength = totalLength;
    tiling.alignNum = 32 / sizeof(float);

    uint32_t coreNum = ascendcPlatform.GetCoreNumAiv();
    uint32_t rawPerCore = totalLength / coreNum;
    tiling.coreCount = (rawPerCore / tiling.alignNum) * tiling.alignNum;
    tiling.coreRemainder = totalLength - tiling.coreCount * coreNum;

    constexpr uint32_t ubSize = 248 * 1024;
    constexpr uint32_t bufferCount = 3;
    constexpr uint32_t pingPong = 2;
    uint32_t maxElements = ubSize / (bufferCount * pingPong * sizeof(float));
    tiling.tileMod = (maxElements / tiling.alignNum) * tiling.alignNum;

    tiling.tileNumPerTime = tiling.tileMod / MASK_FULL;
    tiling.tileCount = tiling.coreCount / tiling.tileMod;
    tiling.tileTail = tiling.coreCount % tiling.tileMod;

    workspaceSize = 0;
}

注意 UB 空间的分配：add 有 3 个队列（A、B、C），每个队列使用双 Buffer（ping-pong），所以 UB 被分成 3 × 2 = 6 块。

8.3.6.2 aclnn 接口注册

aclnnStatus aclnnAddCustomGetWorkspaceSize(
    const aclOpExecutor* executor,
    const aclTensor* a, const aclTensor* b, const aclTensor* c,
    uint64_t* workspaceSize, aclOpExecutor** outExecutor)
{
    AddCustomTilingData tiling;
    uint32_t wsSize = 0;
    AddCustomTilingFunc(
        GetShape(a), GetShape(b), GetShape(c),
        *static_cast<const op::AddCustom*>(executor->op),
        tiling, wsSize);

    *workspaceSize = wsSize;
    SET_TILING(executor, tiling);
    *outExecutor = executor;
    return ACLNN_SUCCESS;
}

aclnnStatus aclnnAddCustomExecute(
    aclrtStream stream, const aclOpExecutor* executor,
    const aclTensor* a, const aclTensor* b, const aclTensor* c,
    void* workspace, uint64_t workspaceSize)
{
    GET_TILING(executor, AddCustomTilingData, tiling);

    uint32_t coreNum = ascendcPlatform.GetCoreNumAiv();
    uint32_t numBlocks = (tiling.totalLength < coreNum) ?
                          tiling.totalLength : coreNum;

    add_custom_kernel<<<numBlocks, nullptr, stream>>>(
        GetAddr(a), GetAddr(b), GetAddr(c), GET_TILING_ADDR(executor));

    return ACLNN_SUCCESS;
}

8.3.7 CMakeLists 配置

算子需要通过 CMakeLists.txt 注册到 CANN 框架：

cmake_minimum_required(VERSION 3.16)

set(OP_NAME "AddCustom")

add_library(${OP_NAME} SHARED
    op_host/add_custom_aclnn.cpp
    op_host/add_custom_tiling.cpp
    op_kernel/add_custom.cpp
)

target_include_directories(${OP_NAME} PRIVATE
    ${CMAKE_CURRENT_SOURCE_DIR}/op_tiling
    ${CANN_TOOLKIT_DIR}/include
    ${CANN_TOOLKIT_DIR}/include/aclnn
)

target_link_libraries(${OP_NAME} PRIVATE
    ascendcl
    acl_op_compiler
)

ascendc_register_op(${OP_NAME}
    OP_TYPE add_custom
    INPUT a tensor float32
    INPUT b tensor float32
    OUTPUT c tensor float32
)

ascendc_register_op 宏将算子的类型名、输入输出参数注册到 CANN 框架中，使得调用方可以通过 aclnnAddCustom 接口调用。

Q&A

Q: add 算子为什么用双 Buffer 而 swap 用单 Buffer？

add 有 Vector 计算阶段（Add API），双 Buffer 可以让 MTE2 搬运和 Vector 计算重叠执行：当 Vector 在处理 ping Buffer 的数据时，MTE2 同时向 pong Buffer 搬运下一批数据。swap 没有 Vector 计算，双 Buffer 无法带来重叠收益。

Q: MASK_FULL 是什么？

MASK_FULL 是 Vector 单元单次操作能处理的最大元素数。对于 float32，MASK_FULL = 128（即 512 字节 / 4 字节）。Add API 的 mask 参数设为 MASK_FULL 表示每次迭代处理 128 个元素。

Q: workspaceSize 为什么设为 0？

add 算子不需要额外的 Workspace 内存（所有中间数据都在 UB 中处理）。Workspace 通常用于需要额外临时存储的复杂算子（如 matmul 的分块中间结果）。

Q: 如何选择直调还是 aclnn 注册？

简单原则：如果算子接口简单（输入输出少、类型固定）、不需要框架自动管理 Workspace 和类型推导，用直调方式更简洁。如果算子需要与 PyTorch/MindSpore 等框架深度集成、需要支持多种数据类型、或需要框架自动管理资源，用 aclnn 注册方式。

Q: repeatParams 中的 {1, 1, 1, 8, 8, 8} 是什么意思？

这是 Add API 的地址步进参数。前三个值 {1, 1, 1} 是 dst/src0/src1 在单次 repeat 内的地址步进（单位为 block），后三个值 {8, 8, 8} 是相邻 repeat 之间的地址步进。对于连续存储的数据，通常设为 {1, 1, 1, 8, 8, 8}。
