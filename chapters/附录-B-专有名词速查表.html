B. 专有名词速查表

Ascend C 开发中涉及的所有专有名词速查，中英文统一按首字母排列。

首字母 术语 英文全称 解释
A ACLAscend Computing Language昇腾计算语言，CANN 提供的基础编程接口
aclblas—ops-blas 仓定义的 BLAS 算子上层接口
aclnn—Ascend C 算子的标准调用接口，前缀为 aclnn 的 C 函数
AI Core—昇腾 NPU 的基本计算单元，包含 Cube / Vector / Scalar 三大计算单元
AICAI CubeAI Core 中负责矩阵乘计算的单元
AIVAI VectorAI Core 中负责向量计算的单元
Ascend C—华为昇腾 NPU 上的 C++ 算子开发框架
ATCAscend Tensor Compiler离线模型编译工具
B 搬运 / 数据搬运Data Movement在 GM 和 UB 之间搬运数据的过程，由 MTE 引擎驱动
搬运引擎Memory Transfer Engine (MTE)负责数据搬运的硬件引擎，包括 MTE1/MTE2/MTE3
BF16Brain Float 1616 位浮点格式，8 位指数 + 7 位尾数，动态范围同 FP32
BufferID—A5 引入的同步机制，硬件自动追踪 Buffer 依赖关系
C CANNCompute Architecture for Neural Networks昇腾统一计算架构
CCEC Cube Engine华为早期底层编程框架，直接操作 Cube / Vector 单元；也指连接上层编程语言和底层硬件的中间表示（CCE 指令）
CCUCollective Communication UnitA5 的通算融合单元，通信不占用 AI Core 算力
乘加运算MAC (Multiply-Accumulate)一次乘法和一次加法的组合运算
重排Data Reordering将数据从一种排布格式转换为另一种格式的过程
存储层次Memory HierarchyGM → L2 → L1 → L0A/L0B/L0C → UB 的多级存储结构
Cube—AI Core 中的矩阵乘加计算单元，16×16×16 MAC 阵列
D Da Vinci 核—昇腾 AI Core 的架构名称
DataCopy—Ascend C 数据搬运核心 API
DataCopyPad—带填充的数据搬运 API，处理不对齐数据
DAV_2201—A2 / A3 芯片的架构号 (NpuArch)
DAV_3510—A5 芯片的架构号 (NpuArch)
对齐Alignment数据地址或长度必须是 32B 的整数倍，否则无法搬运
多核切分Multi-core Partitioning将总数据量分配给多个 AI Core 并行处理的策略
E ELEMENTS_PER_BLOCK—32B 对齐下的最少元素数，FP32 为 8
F 反量化Dequantization将低精度数据（如 INT8）恢复为高精度数据（如 FP32）的过程
填充Padding在数据末尾补齐元素以满足对齐要求
FixPipe—Cube 输出后的后处理单元，支持量化 / 反量化 / Bias 加法
FP8 / FP16 / FP32Float Point 8/16/32浮点数据类型，位宽分别为 8/16/32 位
FracNZ—昇腾 NPU 专用数据排布格式，用于 Transformer QKV 权重
融合算子Fused Operator将多个小算子合并成一个大算子，减少数据搬运开销
G GM / 全局内存Global Memory全局内存，片外 HBM，容量大但延迟高，所有 AI Core 共享
归约Reduction将多个数据聚合为单个数据的操作，如求和、求最大值
H 核函数Kernel Function运行在 NPU AI Core 上的函数，包含 Host 和 Kernel 两部分代码
HiF8—华为自定义 8-bit 浮点格式，指数位和尾数位可配置
Host—CPU 侧代码，负责参数校验、Tiling 计算、启动 Kernel
J 精度标准Precision Standard算子输出与 golden 结果的误差容限，分为 L0/L1/L2 三个等级
K Kernel—运行在 NPU AI Core 上的核函数
L L0A—Cube 左矩阵操作数缓冲区，64KB
L0B—Cube 右矩阵操作数缓冲区，64KB
L0C—Cube 输出缓冲区，A2 为 128KB，A5 为 256KB
L1—每核 512KB 的 Cube 输入缓存
L2—跨核共享缓存
量化Quantization将高精度数据（如 FP32）转换为低精度数据（如 INT8）的过程
流水线Pipeline搬运和计算重叠执行的并行策略，提升效率
M mask—Vector 指令的位掩码，控制哪些元素参与计算
脉动阵列Systolic ArrayCube 的核心结构，数据像波浪一样流过乘加单元
MemBase—SIMD 编程子模型，中间数据存储在 UB 中
模拟执行Simulation在无 NPU 硬件的环境下模拟算子执行，用于功能验证
MPMDMultiple Program Multiple Data多程序多数据，不同核运行不同代码
MTE / MTE1 / MTE2 / MTE3Memory Transfer Engine数据搬运引擎，MTE1 核内搬运，MTE2 从 GM 搬入，MTE3 搬出到 GM
N NC1HWC05HD昇腾 NPU 专用数据排布格式，用于卷积输入
NCHW / NHWC / ND—数据排布格式，分别对应 PyTorch 默认、TensorFlow 默认、最通用格式
NDDMAN-Dimensional DMAA5 新增的高维 DMA 指令
O opbase—所有算子仓依赖的基础框架库
ops-blas / ops-math / ops-nn—算子代码仓，分别对应线性代数、数学运算、神经网络算子
P Pipe—执行单元的指令流水线
PyPTOParallel Tensor/Tile OperationMPMD 编程框架
Q 切分策略Tiling Strategy决定每个 AI Core 处理哪些数据的策略，包括多核切分、UB 切分、循环切分
R RegBase—SIMD 编程子模型（A5 新增），中间数据存储在寄存器中
repeat—Vector 指令的重复执行机制
S 算子Operator对张量执行的数学运算，包含 Host 和 Kernel 两部分代码，调度 NPU 完成计算
Scalar—AI Core 中负责控制流和地址计算的单元
数据排布Data Layout多维张量在内存中的存储顺序，如 NCHW、NHWC、ND 等
SIMDSingle Instruction Multiple Data单指令多数据编程模型
SIMTSingle Instruction Multiple Threads单指令多线程编程模型（A5 新增）
S（续） SPMDSingle Program Multiple Data单程序多数据，Ascend C 默认多核模型
sswap—BLAS Level-1 向量交换算子
同步Synchronization协调不同执行单元（Pipe）之间的依赖关系，如 SetFlag/WaitFlag、BufferID
T TBETensor Boost Engine基于 Python DSL + TVM 的编程框架
Tiling / TilingData—数据切分策略及对应的结构体，决定每个 AI Core 处理哪些数据
TPipe / TQue / TQueBind—Ascend C 流水线和队列管理工具，TQueBind 支持输入输出共用
统一缓冲区Unified Buffer (UB)每个 AI Core 的片上高速缓存，用于 Vector 计算
U UBUnified Buffer统一缓冲区，Vector 计算的主战场，A5 为 248KB
V Vector—AI Core 中的向量计算单元，256 Bytes 宽
向量计算Vector Computation对一组数据同时执行相同操作的计算方式
W Warp—SIMT 模式下 32 线程的分组单位
尾块Tail Block数据切分后剩余的部分，通常分配给最后一个核处理
伪量化Fake Quantization训练时模拟量化误差的技术，前向做量化→反量化，反向用 STE
