B. 专有名词速查表

Ascend C 开发中涉及的所有专有名词速查，按字母顺序排列。

术语 英文全称 解释
ACLAscend Computing Language昇腾计算语言，CANN 提供的基础编程接口
aclblas—ops-blas 仓定义的 BLAS 算子上层接口
aclnn—Ascend C 算子的标准调用接口，前缀为 aclnn 的 C 函数
AI Core—昇腾 NPU 的基本计算单元，包含 Cube / Vector / Scalar 三大计算单元
AICAI CubeAI Core 中负责矩阵乘计算的单元
AIVAI VectorAI Core 中负责向量计算的单元
Ascend C—华为昇腾 NPU 上的 C++ 算子开发框架
ATCAscend Tensor Compiler离线模型编译工具
BF16Brain Float 1616 位浮点格式，8 位指数 + 7 位尾数，动态范围同 FP32
BufferID—A5 引入的同步机制，硬件自动追踪 Buffer 依赖关系
CANNCompute Architecture for Neural Networks昇腾统一计算架构
CCEC Cube Engine华为早期底层编程框架，直接操作 Cube / Vector 单元
CCE 指令—连接上层编程语言和底层硬件的中间表示
CCUCollective Communication UnitA5 的通算融合单元，通信不占用 AI Core 算力
Cube—AI Core 中的矩阵乘加计算单元，16×16×16 MAC 阵列
Da Vinci 核—昇腾 AI Core 的架构名称
DataCopy—Ascend C 数据搬运核心 API
DataCopyPad—带填充的数据搬运 API，处理不对齐数据
DAV_2201—A2 / A3 芯片的架构号 (NpuArch)
DAV_3510—A5 芯片的架构号 (NpuArch)
ELEMENTS_PER_BLOCK—32B 对齐下的最少元素数，FP32 为 8
FixPipe—Cube 输出后的后处理单元，支持量化 / 反量化 / Bias 加法
FP8Float Point 88 位浮点格式，A5 新增
FP16Float Point 1616 位浮点格式，5 位指数 + 10 位尾数
FP32Float Point 3232 位浮点格式，8 位指数 + 23 位尾数
FracNZ—昇腾 NPU 专用数据排布格式，用于 Transformer QKV 权重
GMGlobal Memory全局内存，片外 HBM，容量大但延迟高
HiF8—华为自定义 8-bit 浮点格式，指数位和尾数位可配置
Host—CPU 侧代码，负责参数校验、Tiling 计算、启动 Kernel
Kernel—运行在 NPU AI Core 上的核函数
L0A—Cube 左矩阵操作数缓冲区，64KB
L0B—Cube 右矩阵操作数缓冲区，64KB
L0C—Cube 输出缓冲区，A2 为 128KB，A5 为 256KB
L1—每核 512KB 的 Cube 输入缓存
L2—跨核共享缓存
MACMultiply-Accumulate乘加运算
mask—Vector 指令的位掩码，控制哪些元素参与计算
MemBase—SIMD 编程子模型，中间数据存储在 UB 中
MPMDMultiple Program Multiple Data多程序多数据，不同核运行不同代码
MTEMemory Transfer Engine数据搬运引擎
MTE1—核内搬运引擎（L1↔L0、L1↔UB）
MTE2—从 GM 搬入的引擎
MTE3—搬出到 GM 的引擎
MXFP4Microscaling Float Point 4微缩 4-bit 浮点格式，32 个数据共享 1 个 Scale
MXFP8Microscaling Float Point 8微缩 8-bit 浮点格式，32 个数据共享 1 个 Scale
NC1HWC05HD昇腾 NPU 专用数据排布格式，用于卷积输入
NCHW—数据排布格式，PyTorch 默认
ND—最通用的数据排布格式，任意维度
NDDMAN-Dimensional DMAA5 新增的高维 DMA 指令
NHWC—数据排布格式，TensorFlow 默认
NPUNeural Processing Unit神经网络处理器
opbase—所有算子仓依赖的基础框架库
ops-blas—线性代数算子仓
ops-math—数学运算算子仓
ops-nn—神经网络算子仓
Pipe—执行单元的指令流水线
PyPTOParallel Tensor/Tile OperationMPMD 编程框架
RegBase—SIMD 编程子模型（A5 新增），中间数据存储在寄存器中
repeat—Vector 指令的重复执行机制
Scalar—AI Core 中负责控制流和地址计算的单元
SIMDSingle Instruction Multiple Data单指令多数据编程模型
SIMTSingle Instruction Multiple Threads单指令多线程编程模型（A5 新增）
SPMDSingle Program Multiple Data单程序多数据，Ascend C 默认多核模型
sswap—BLAS Level-1 向量交换算子
TBETensor Boost Engine基于 Python DSL + TVM 的编程框架
Tiling—数据切分策略，决定每个 AI Core 处理哪些数据
TilingData—Host 和 Kernel 之间的通信协议结构体
TPipe—Ascend C 流水线管理工具
TQue—Ascend C 队列管理工具
TQueBind—输入输出共用队列的队列类型
UBUnified Buffer统一缓冲区，Vector 计算的主战场，A5 为 248KB
Vector—AI Core 中的向量计算单元，128 位宽
Warp—SIMT 模式下 32 线程的分组单位
