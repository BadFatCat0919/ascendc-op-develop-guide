4.2.1 获取 swap 算子代码

swap 是 BLAS（Basic Linear Algebra Subprograms）标准中最简单的算子之一：交换两个向量的对应元素。我们用它作为第一个实战项目，因为它逻辑简单但结构完整，能让你快速理解算子开发的全流程。

克隆 ops-blas 仓库

# 克隆 ops-blas 算子仓库
git clone https://gitcode.com/cann/ops-blas.git
cd ops-blas

# 查看仓库结构
ls -la
# 输出：
# blas/          # 算子源码目录
# build.sh       # 编译脚本
# test/          # 测试用例
# CMakeLists.txt # CMake 配置
# ...

定位 swap 算子代码

# swap 算子位于 blas/swap/ 目录
ls blas/swap/
# 输出：
# arch35/        # 针对 A5 架构的实现
# CMakeLists.txt

# 查看 arch35 目录下的文件
ls blas/swap/arch35/
# 输出：
# swap_host.cpp       # Host 侧实现
# swap_kernel.cpp     # Kernel 侧实现
# swap_tiling_data.h  # Tiling 数据结构定义

代码文件说明

文件作用运行位置
swap_host.cpp参数校验、Tiling 计算、启动 KernelCPU（Host 侧）
swap_kernel.cppAscend C 核函数实现NPU（Device 侧）
swap_tiling_data.hHost 和 Kernel 共享的 Tiling 数据结构编译时共用

4.2.2 编译 swap 算子

ops-blas 仓库提供了统一的编译脚本 build.sh，一行命令就能编译指定算子。

编译命令

# 编译 swap 算子，目标芯片为 Ascend 910B（架构代号 ascend950）
bash build.sh --ops=swap --soc=ascend950

# 编译过程输出（简化）：
# [INFO] Building swap for ascend950...
# [INFO] Compiling swap_kernel.cpp...
# [INFO] Compiling swap_host.cpp...
# [INFO] Linking...
# [INFO] Build success!
# [INFO] Output: out/swap/libswap.so

编译参数说明

参数说明示例值
--ops要编译的算子名称swap、gemm、axpy
--soc目标芯片型号ascend950（910B）、ascend910
--debug开启调试模式添加此标志
--clean清理后重新编译添加此标志

编译产物

# 编译成功后，产物在 out/ 目录
ls out/swap/
# 输出：
# libswap.so           # 动态库（算子实现）
# swap_kernel.o        # Kernel 目标文件
# swap_host.o          # Host 目标文件

# 同时会生成编译日志
cat out/swap/build.log

4.2.3 运行测试

ops-blas 仓库自带 ST（System Test）测试框架，可以自动验证算子的正确性。

运行 ST 测试

# 运行 swap 算子的 ST 测试
cd test/swap
bash run_test.sh

# 或者使用统一测试入口
cd ../..
bash test/run_all_tests.sh --ops=swap

测试用例说明

4.2.5 理解输出日志

编译和运行过程中会产生大量日志，学会快速定位关键信息很重要。

编译日志关键字段

# 编译成功的关键标志
[INFO] Build success!

# 如果编译失败，关注 error 信息
[ERROR] swap_kernel.cpp:42: undefined reference to 'DataCopy'
# 说明：API 使用错误，检查头文件包含

[ERROR] swap_kernel.cpp:15: expected ';' after expression
# 说明：语法错误，检查代码

运行日志关键字段

日志级别含义处理方式
[INFO]正常信息一般无需关注
[WARN]警告信息检查是否影响功能
[ERROR]错误信息必须修复
[FATAL]致命错误程序终止，必须修复

常见错误码

# 运行时错误码示例
[ERROR] ACL Error: 161001
# 含义：Tiling 参数错误
# 解决：检查 TilingData 的计算逻辑

[ERROR] ACL Error: 361001
# 含义：Kernel 执行超时
# 解决：检查 Kernel 是否有死循环或同步问题

[ERROR] ACL Error: 561001
# 含义：内存分配失败
# 解决：检查 UB 空间是否超限

4.2.6 修改代码重新编译运行

现在你已经跑通了 swap 算子，让我们尝试修改代码并重新运行。

练习：添加调试日志

# 1. 打开 swap_kernel.cpp
vim blas/swap/arch35/swap_kernel.cpp

# 2. 在 Process 函数中添加日志
# 找到 Process() 函数，添加：
printf("Kernel started, block_idx=%d, totalN=%d\n", block_idx, tiling.totalN);

# 3. 重新编译（增量编译，速度更快）
bash build.sh --ops=swap --soc=ascend950

# 4. 重新运行测试
cd test/swap && bash run_test.sh

# 5. 查看日志中的新输出
grep "Kernel started" plog/plog-*.log

练习：修改数据规模

# 1. 打开测试配置文件
vim test/swap/swap_test.csv

# 2. 添加一行新的测试用例
# 格式：dtype,N,incx,incy
# 添加：float32,8192,1,1

# 3. 重新运行测试
bash run_test.sh

# 4. 查看新测试用例的结果
cat results/swap_test_report.txt | grep 8192

增量编译 vs 全量编译

场景命令耗时
只修改了 Kernel 代码bash build.sh --ops=swap --soc=ascend950~10 秒
修改了 CMakeLists.txtbash build.sh --ops=swap --soc=ascend950 --clean~30 秒
切换了目标芯片bash build.sh --ops=swap --soc=ascend910 --clean~30 秒
