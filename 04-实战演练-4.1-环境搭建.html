4.1.1 CANN Toolkit 安装（Linux 环境）

算子开发的第一步是安装 CANN Toolkit。CANN（Compute Architecture for Neural Networks）是昇腾 NPU 的统一计算架构，Toolkit 里包含了编译器、运行时库和各种开发工具。

下载 CANN Toolkit

访问昇腾社区官网下载页面：

# 访问下载页面
https://www.hiascend.com/software/cann/community

# 选择对应版本（推荐最新稳定版）
# 下载文件名格式：Ascend-cann-toolkit_8.x.x_linux-aarch64.run（ARM）
# 或：Ascend-cann-toolkit_8.x.x_linux-x86_64.run（x86）

根据你的服务器架构选择对应的安装包：

架构适用场景安装包后缀
aarch64ARM 服务器（如 Atlas 800）linux-aarch64.run
x86_64x86 服务器 + NPU 加速卡linux-x86_64.run

安装步骤

# 1. 添加执行权限
chmod +x Ascend-cann-toolkit_8.x.x_linux-aarch64.run

# 2. 执行安装（默认安装到 ~/Ascend/ascend-toolkit）
./Ascend-cann-toolkit_8.x.x_linux-aarch64.run --install

# 3. 如果需要指定安装路径
./Ascend-cann-toolkit_8.x.x_linux-aarch64.run --install --install-path=/opt/Ascend

# 4. 安装完成后，目录结构如下
~/Ascend/ascend-toolkit/latest/
├── bin/              # 可执行工具
├── include/          # 头文件
├── lib64/            # 动态库
├── toolkit/tools/    # 开发工具
└── set_env.sh        # 环境变量脚本（重要！）

安装依赖

CANN Toolkit 依赖一些系统库，确保已安装：

# Ubuntu/Debian
sudo apt-get install -y gcc g++ make cmake python3 python3-pip

# CentOS/RHEL
sudo yum install -y gcc gcc-c++ make cmake python3 python3-pip

# 安装 Python 依赖（用于算子开发工具链）
pip3 install numpy decorator sympy cffi pyyaml pathlib psutil

4.1.2 环境变量配置

安装完成后，必须配置环境变量，否则编译器找不到头文件和库。

临时生效（当前终端）

# 执行环境变量脚本
source ~/Ascend/ascend-toolkit/latest/set_env.sh

# 验证环境变量是否生效
echo $ASCEND_HOME
# 应输出：/home/你的用户名/Ascend/ascend-toolkit/latest

# 检查 PATH 是否包含 CANN 工具
which ascendc
# 应输出：/home/你的用户名/Ascend/ascend-toolkit/latest/toolkit/tools/ascendc

永久生效（推荐）

# 将 source 命令添加到 ~/.bashrc
echo 'source ~/Ascend/ascend-toolkit/latest/set_env.sh' >> ~/.bashrc

# 重新加载配置
source ~/.bashrc

# 或者添加到 ~/.profile（适用于登录 shell）
echo 'source ~/Ascend/ascend-toolkit/latest/set_env.sh' >> ~/.profile

关键环境变量说明

变量名作用典型值
ASCEND_HOMECANN 安装根目录~/Ascend/ascend-toolkit/latest
PATH可执行文件搜索路径包含 $ASCEND_HOME/bin
LD_LIBRARY_PATH动态库搜索路径包含 $ASCEND_HOME/lib64
ASCEND_OPP_PATH算子包路径$ASCEND_HOME/opp

4.1.3 验证安装

安装完成后，运行以下命令验证：

# 1. 检查 ascendc 编译器版本
ascendc --version
# 输出示例：Ascend C Compiler version 8.x.x

# 2. 检查编译器路径
which ascendc
# 应输出 CANN 安装目录下的路径

# 3. 检查运行时库
ls $ASCEND_HOME/lib64/ | grep libascendcl
# 应看到 libascendcl.so 等文件

# 4. 检查 NPU 设备（如果有硬件）
npu-smi info
# 应显示 NPU 设备信息表格

4.1.4 编译器配置

Ascend C 使用专用的编译器 ascendc，它基于 LLVM/Clang 开发，支持将 C++ 代码编译成 NPU 可执行的二进制。

编译器基本用法

# 编译单个 Kernel 文件
ascendc -c -O2 -o kernel.o kernel.cpp

# 编译选项说明
# -c          : 只编译，不链接
# -O2         : 优化级别（O0/O1/O2/O3）
# -o          : 输出文件名
# --target    : 目标架构（如 ascend910b）

# 完整编译命令示例
ascendc -c -O2 --target=ascend910b -o swap_kernel.o swap_kernel.cpp

常用编译选项

选项说明示例
--target目标芯片架构--target=ascend910b
-I头文件搜索路径-I$ASCEND_HOME/include
-D宏定义-DENABLE_DEBUG
-g生成调试信息-g（配合仿真器使用）
-stdC++ 标准-std=c++17

4.1.5 仿真器使用

如果没有 NPU 硬件，可以使用 CANN Simulator（仿真器）来验证算子功能。

安装仿真器

# 仿真器通常随 CANN Toolkit 一起安装
# 检查是否已安装
ls $ASCEND_HOME/simulator/

# 如果没有，单独下载安装
# 从昇腾社区下载 CANN Simulator 安装包
./Ascend-cann-simulator_8.x.x_linux-aarch64.run --install

使用仿真器运行算子

# 1. 编译时添加仿真器支持
ascendc -c -g --sim-mode -o kernel.o kernel.cpp

# 2. 链接生成可执行文件
ascendc -o test_sim test_main.cpp kernel.o -lsim_ascend

# 3. 运行仿真
./test_sim

# 4. 查看仿真输出
# 仿真器会在当前目录生成 sim_output/ 目录
# 包含算子的输入输出数据（二进制格式）

仿真器 vs 真实硬件

特性仿真器真实 NPU
功能验证✅ 支持✅ 支持
精度验证✅ 支持✅ 支持
性能测试❌ 不准确✅ 准确
调试便利✅ 可单步调试❌ 需 plog
运行速度慢（CPU 模拟）快（硬件加速）

4.1.6 常见问题排查

问题 1：ascendc 命令找不到

# 现象
ascendc --version
# bash: ascendc: command not found

# 原因：环境变量未配置
# 解决：
source ~/Ascend/ascend-toolkit/latest/set_env.sh

# 验证
echo $PATH | grep ascend

问题 2：编译报错找不到头文件

# 现象
fatal error: acl/acl.h: No such file or directory

# 原因：ASCEND_HOME 未正确设置
# 解决：
echo $ASCEND_HOME
# 如果为空，重新 source set_env.sh

# 或者手动指定头文件路径
ascendc -c -I$ASCEND_HOME/include -o kernel.o kernel.cpp

问题 3：npu-smi 显示设备离线

# 现象
npu-smi info
# NPU 状态显示 Offline 或 Not Ready

# 解决步骤：
# 1. 检查驱动是否加载
lsmod | grep davinci

# 2. 重新加载驱动
sudo rmmod davinci
sudo modprobe davinci

# 3. 重启 NPU 设备
sudo npu-smi set -t reset -i 0

# 4. 如果仍不行，重启服务器

问题 4：仿真器运行报错

# 现象
./test_sim
# error: libsim_ascend.so: cannot open shared object file

# 原因：仿真器库路径未添加到 LD_LIBRARY_PATH
# 解决：
export LD_LIBRARY_PATH=$ASCEND_HOME/simulator/lib64:$LD_LIBRARY_PATH

# 添加到 ~/.bashrc 永久生效
echo 'export LD_LIBRARY_PATH=$ASCEND_HOME/simulator/lib64:$LD_LIBRARY_PATH' >> ~/.bashrc

问题 5：Python 版本冲突

# 现象
pip3 install numpy
# 安装到了 Python 3.6，但 CANN 需要 Python 3.9+

# 解决：使用指定版本的 pip
python3.9 -m pip install numpy decorator sympy cffi pyyaml

# 或者创建虚拟环境
python3.9 -m venv ~/cann_env
source ~/cann_env/bin/activate
pip install numpy decorator sympy cffi pyyaml pathlib psutil
